# Stargazer 磨き込みプラン — 全領域を9/10へ

## 現状の正直な評価
- 「改善した」と報告しても、別セッションで検証すると点数が下がる
- 原因: 表面的な修正（import追加、コメント変更）で「完了」としていた
- 本質: ユーザーが触って体感が変わるレベルの変更が不足

## 原則
1. **ユーザーが触って違いが分かる変更のみ**をカウントする
2. 「APIを作った」だけでは0点。フロントから呼ばれ、UIに表示され、データが流れて初めて1点
3. 各領域の具体的なベンチマーク行動を定義し、それが実現できているか検証する

---

## A. FTUE (4→9): 初回体験の魔法

### ベンチマーク行動: 「2分以内にユーザーが『おっ』と思う」
### 具体施策:

1. **5問回答後のライブタイプ・プレビュー**
   - InitialOnboardingFlow.tsx の5問目完了時に、途中経過の星座タイプを表示
   - 「あなたは○○型に近づいています...」のリアルタイム表示
   - 全問回答を待たず、途中の軸スコアから仮タイプを算出

2. **段階的プログレスバー（今どこにいるか明示）**
   - 「Step 1/4: あなたの傾向を知る」→「Step 2/4: 判断パターン」→...
   - 各ステップ完了時にミニ成果物を見せる

3. **初回完了時の演出強化**
   - タイプ確定時: 星座が組み上がるアニメーション（3秒）
   - 「あなたの星座が見つかりました」→ タイプカード表示
   - 初回インサイト1つを即座に生成して提示

### 検証基準:
- [ ] 5問回答後に仮タイプが表示されるか
- [ ] 全問完了時にタイプ確定アニメーションが再生されるか
- [ ] 完了直後にパーソナライズされたインサイトが1つ表示されるか

## B. 日次エンゲージメント (5→9): 毎日戻りたくなる

### ベンチマーク行動: 「ユーザーが朝起きてアプリを開く理由がある」
### 具体施策:

   - 全画面で表示（タブ切り替えても消えない）

2. **デイリーXPスコアシステム**
   - 朝の一問回答: +30pt
   - 観測完了: +50pt
   - 予測検証: +20pt
   - Vanishing Insight閲覧: +10pt
   - 反応ボタン押下: +5pt
   - 合計を「今日の観測スコア」として表示

3. **ストリーク断裂の感情的フック**
   - 1日サボり: 「Alterがあなたのことを考えています」
   - 3日サボり: 精度低下グラフ + 「精度が○%下がりました」
   - 7日サボり: 「あなたの星座が薄くなっています」（視覚的に星が暗くなる）

4. **曜日別コンテンツローテーション**
   - 月: 週間サマリー公開
   - 水: Blind Spot配信
   - 金: 予測精度レポート
   - 日: Alter手紙配信タイミング

### 検証基準:
- [ ] ホーム画面上部にストリーク+スコアが常時表示されているか
- [ ] 各アクションでスコアが加算されるか

## C. 手触り (6→9): 触ること自体が心地よい

### ベンチマーク行動: 「ボタンを押すたびに小さな快感がある」
### 具体施策:

1. **アニメーション速度の全面見直し**
   - 現在: 350-800ms → 目標: 180-300ms
   - カード登場: 350ms → 220ms
   - タブ切り替え: 300ms → 180ms
   - ボタンフィードバック: 即時（60ms以下）
   - 劇的演出（タイプ確定等）のみ600ms以上を許可

2. **ハプティクス接続**
   - 回答確定: medium haptic
   - インサイト表示: light haptic
   - 予測的中: heavy haptic + confetti
   - ストリーク更新: notification haptic

3. **ローディングスケルトンの実装**
   - 各セクション固有のスケルトン（汎用スピナー禁止）
   - TodaySummary用、RadarChart用、InsightCard用
   - コンテンツ表示時: スケルトン→実データのクロスフェード

4. **スクロールパフォーマンス最適化**
   - contain: paint を主要スクロールコンテナに追加
   - will-change: transform をアニメーション要素に追加
   - Intersection Observer で画面外コンポーネントをアンマウント

### 検証基準:
- [ ] 全アニメーションが300ms以下か（劇的演出除く）
- [ ] 回答時にハプティクスが発火するか
- [ ] ローディング中にセクション固有のスケルトンが表示されるか

## D. 情報設計 (5→9): 迷わない導線

### ベンチマーク行動: 「初見で3タップ以内に目的地に着ける」
### 具体施策:

1. **ホーム画面の優先度整理**

3. **TraitsTab のデフォルト簡素化**
   - 初期表示: 変化が大きいTop 5軸のみ
   - 「全45軸を見る」ボタンで展開
   - 各軸に1行の解説テキスト

4. **パンくずナビゲーション**
   - Deep機能の各画面に「← 深層」の戻りリンク
   - 現在地インジケーター

### 検証基準:
- [ ] ホーム画面のファーストビューに主要CTAが1つだけ表示されるか
- [ ] Deep機能のインデックスが存在し、全機能が一覧できるか
- [ ] TraitsTabで初期表示が5軸以下か

## J. パーソナライゼーション (5→9): 自分だけの体験

### 具体施策:

1. **反応フィードバックの学習ループ**
   - 「響いた」が多い軸 → その軸のインサイトを増やす
   - 「ピンとこない」が多いカテゴリ → 出現頻度を下げる
   - reaction データを次回AI生成のcontextに含める

2. **インサイトの出典表示**
   - 各インサイトに「○○の矛盾スコアに基づく」の表示
   - ユーザーが「なぜこれが出たか」を理解できる

3. **朝の質問の適応的選択**
   - 最近矛盾が検出された軸に関する質問を優先
   - 回答が少ない軸の質問を出す
   - 同じ質問の繰り返しを30日間禁止

### 検証基準:
- [ ] 反応データが次回のAI生成contextに含まれているか
- [ ] インサイトに出典が表示されているか
- [ ] 朝の質問が適応的に選択されているか
