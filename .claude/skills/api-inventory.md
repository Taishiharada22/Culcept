# API Inventory — APIルート棚卸し

Build Unit の CTO として、APIルートの棚卸しを実行してください。

## 手順

1. `app/api/` 配下の全 `route.ts` ファイルを検索
2. 各ファイルについて以下を確認:
   - HTTP メソッド（GET / POST / PUT / DELETE）
   - 認証チェックの有無（`supabase.auth.getUser()` 等）
   - try-catch によるエラーハンドリングの有無
   - admin チェックの有無（`isAdmin` 等）
3. 結果をテーブル形式でまとめる

## 出力フォーマット

```
## API 棚卸し [今日の日付]

### サマリー
- 総エンドポイント数: [N]
- 認証あり: [N] / 認証なし: [N]
- エラーハンドリングあり: [N] / なし: [N]

### 要注意エンドポイント
[認証なし or エラーハンドリングなしのものをリスト]

| パス | メソッド | 認証 | エラー処理 | 備考 |
|------|---------|------|-----------|------|
| /api/xxx | GET | :x: | :x: | 要修正 |

### カテゴリ別
| カテゴリ | エンドポイント数 | 備考 |
|---------|----------------|------|
| stargazer | N | ... |
| battle | N | ... |
| ... | ... | ... |
```

## 注意
- 全件が多い場合は「要注意」のみ詳細表示、正常なものはサマリーのみ
- 修正提案は別途 `/bug-triage` で対応
