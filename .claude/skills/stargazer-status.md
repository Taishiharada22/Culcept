# Stargazer Status — 機能完成度ダッシュボード

Product Unit の PM として、Stargazer 機能群の完成度を一括スキャンしてください。

## 手順

1. 以下を**並列で**スキャン:
   - `app/stargazer/` 配下の全ページ（`page.tsx`）とコンポーネント一覧
   - `app/api/stargazer/` 配下の全 API ルート（`route.ts`）
   - `app/api/cron/stargazer-*` 配下の cron ジョブ
   - `supabase/migrations/*stargazer*` のマイグレーションファイル
   - コード内の `TODO`, `FIXME`, `HACK` コメント（stargazer 関連ファイルのみ）

2. 各サブ機能について以下を判定:
   - **ページ**: 存在するか、空でないか
   - **API**: 対応する API ルートがあるか
   - **DB**: 必要なマイグレーションがあるか
   - **完成度**: コードの実装状態（スケルトン / 部分実装 / 完成）

3. 以下のフォーマットで報告

## 出力フォーマット

```
## Stargazer ステータス [今日の日付]

### 全体進捗: [完成数] / [総機能数] （[%]）

### 機能別ステータス

| サブ機能 | ページ | API | DB | 完成度 | 備考 |
|---------|--------|-----|-----|--------|------|
| コア観測 | ✅/❌ | ✅/❌ | ✅/❌ | 🟢/🟡/🔴 | ... |
| ... | ... | ... | ... | ... | ... |

### 未実装・要対応
1. [最重要の未完成機能]
2. ...

### TODO/FIXME 残数
- 総数: [N]
- 重要なもの:
  - [file:line] [内容]

### 完成までに必要な作業（推定）
1. ...
2. ...
3. ...

### Aneurasync 設計思想との整合
- [ ] 深層観測の体験が一貫しているか
- [ ] 「自分って、そういう人間だったのか」に到達できる導線があるか
```

## 注意
- スキャンのみ。修正は CEO 承認後
- コンポーネント数が多い場合はサブ機能単位でグルーピングして報告
- 判断に迷う場合は「要確認」として CEO に提示
