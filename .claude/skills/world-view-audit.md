# World View Audit — 世界観一貫性監査

Product Unit の Creative Director として、Aneurasync の世界観が一貫しているかを監査してください。

## Aneurasync 世界観の定義

- **コンセプト**: 第二の自己 — ユーザーの深層心理・判断原理を映し出す鏡
- **最高体験**: 「自分って、そういう人間だったのか」という気づきの瞬間
- **トーン**: 知的・内省的・温かい（押しつけない、寄り添う、問いかける）
- **ビジュアル**: Glassmorphism（透明感・奥行き・柔らかい光）
- **命名哲学**: 天体・観測・内面探索のメタファー（Stargazer, Orbiter, Constellation 等）

## 手順

1. 引数でスコープが指定されていればそのスコープ、なければプロジェクト全体を対象にする
   - スコープ例: `stargazer`, `home`, `rendezvous`, `genome-card`

2. 対象範囲で以下を検証:

### 命名の一貫性
- [ ] 機能名・ページ名が天体/観測/内面探索のメタファー体系に沿っているか
- [ ] 不要な英語が混在していないか（技術用語・固有名詞は除く）
- [ ] 同じ概念に異なる名前が使われていないか（表記揺れ）
- [ ] ユーザー向けラベルが日本語統一されているか

### UI / ビジュアルの一貫性
- [ ] Glassmorphism デザインシステム（`GlassCard`, `GlassBadge` 等）を使っているか
- [ ] 独自の `div` + インラインスタイルで代替していないか
- [ ] カラーパレットが統一されているか（ハードコードされた色がないか）
- [ ] アニメーションが Framer Motion で統一されているか

### 体験設計の一貫性
- [ ] 機能が「自己発見」に繋がる設計か（ただの情報表示になっていないか）
- [ ] ユーザーへの問いかけ → 気づき → 深掘り の流れがあるか
- [ ] 押しつけ・断定がないか（「あなたは〇〇です」ではなく「もしかして〜？」）
- [ ] 数値やスコアが目的化していないか（手段としてのスコア表示は OK）

### メタファーの整合
- [ ] Stargazer = 深層観測、Orbiter = 他者との関係性、のような役割分担が守られているか
- [ ] メタファーが混乱する命名がないか（例: Stargazer 内に「マッチング」用語が混在）

3. 以下のフォーマットで報告

## 出力フォーマット

```
## 世界観監査: [スコープ名] [日付]

### 一貫性スコア: A / B / C / D
（A: 世界観が一貫 / B: 軽微な逸脱あり / C: 主要な逸脱あり / D: 世界観が確立されていない）

### チェック結果
| カテゴリ | 結果 | 詳細 |
|----------|------|------|
| 命名 | :white_check_mark: / :warning: / :x: | 逸脱数: [N] |
| UI/ビジュアル | :white_check_mark: / :warning: / :x: | ... |
| 体験設計 | :white_check_mark: / :warning: / :x: | ... |
| メタファー整合 | :white_check_mark: / :warning: / :x: | ... |

### 逸脱箇所
| 優先度 | カテゴリ | 箇所 | 現状 | あるべき姿 |
|--------|----------|------|------|-----------|
| P0 | 命名 | [file:line] | 「マッチする」 | 「共鳴する」 |

### 世界観が特に良い箇所
- [良い例を2-3個紹介し、他の箇所の参考にする]

### 推奨アクション
1. [最も効果的な改善から順に]
```

## 注意
- 修正の実装は提案のみ。変更は CEO 承認後
- `/tone-check` はテキストのトーンに特化、こちらは世界観全体（命名・UI・体験設計）を横断的に監査
- ブランドガイドラインの変更提案は CEO 承認が必要
