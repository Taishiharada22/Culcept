# Tone Check — トーン一貫性監査

Growth Unit の Brand Strategist として、UIテキストのトーン監査を行ってください。

## 手順

1. 引数でページが指定されていればそのページ、なければ直近変更のページを対象にする
2. 対象ページの `.tsx` ファイルを読み、以下のテキスト要素を抽出:
   - ページタイトル・見出し
   - ボタンラベル
   - 説明文・プレースホルダー
   - 空状態（Empty State）のメッセージ
   - エラーメッセージ・トースト
   - ツールチップ・ヘルプテキスト

3. 以下の基準でチェック:

### Aneurasync トーン基準

**必須**
- [ ] 日本語で統一されているか（技術用語・固有名詞を除く）
- [ ] 「知的・内省的・温かい」トーンか
- [ ] ユーザーの自己発見を促す言葉選びか

**NG パターン**
- 煽り・焦らせる表現（「今すぐ」「急いで」「限定」）
- 過度にカジュアルな表現（「ヤバい」「マジで」）
- 過度な絵文字の使用
- 命令口調（「〜しろ」「〜しなさい」）
- 英語の不要な混在

**推奨パターン**
- 問いかけ型（「あなたは〜？」「もしかして〜」）
- 発見を促す表現（「気づき」「発見」「あなたらしさ」）
- 寄り添う表現（「一緒に」「少しずつ」）
- 内省を誘う表現（「ふと思う」「振り返ると」）

4. 以下のフォーマットで報告

## 出力フォーマット

```
## トーン監査: [ページ名]

### 総合スコア: A / B / C / D

### チェック結果
| 項目 | 結果 | 詳細 |
|------|------|------|
| 日本語統一 | ✅/⚠️/❌ | ... |
| トーン整合 | ✅/⚠️/❌ | ... |
| NG パターン | ✅/⚠️/❌ | 検出数: [N] |
| 推奨パターン活用 | ✅/⚠️/❌ | ... |

### 逸脱箇所と修正案
| 箇所 | 現在のテキスト | 問題 | 修正案 |
|------|--------------|------|--------|
| [file:line] | 「...」 | [NG理由] | 「...」 |

### 良い例（このページの中で）
- [トーンが良いテキストがあれば紹介]
```

## 注意
- 修正案は提案のみ。変更は CEO 承認後
- 固有名詞（Stargazer, Aneurasync 等）は英語のまま OK
- 技術的なエラーメッセージ（404, 500 等）は対象外
- `/ux-audit` との違い: こちらはテキスト・コピーに特化。デザインシステム準拠は `/ux-audit` の範囲
