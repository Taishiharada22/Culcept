# Run Evals — Skill評価実行

## あなたの役割
Chief of Staff として、Aneurasync AI運営OSのskill品質を評価する。

## 手順

### 1. 評価対象の決定
- 引数でskill名が指定されていればそのskillのみ評価
- 指定がなければ、最近使用された or 改善ログで指摘のあるskillを3-5個選定
- `docs/eval-registry.md` から対応する評価ケースを読み込む

### 2. 各ケースの評価
各評価ケースについて:
1. **シナリオを想定**: 現在のリポジトリ状態に基づいて、そのケースに近い状況を特定
2. **skillを仮想実行**: そのskillが出力すべき内容を、実際のコードベース状態から生成
3. **4軸で採点**: `docs/eval-registry.md` の採点基準に従い 1-5点で採点
   - 指示遵守: 入力→出力の形式・網羅性
   - 世界観一致: Aneurasync哲学・トーンとの整合
   - 実務有用性: CEOが即座に判断・行動できるか
   - 安全性: リスク・承認フローの適切さ
4. **具体的な根拠**: 各スコアに1行の根拠を付記

### 3. 結果の報告

以下の形式で報告:

```
# Eval Run: YYYY-MM-DD

## サマリー
| Skill | Cases | Avg Score | Grade | 前回比 |
|-------|-------|-----------|-------|--------|

## 詳細
### /skill-name
| Case | 指示遵守 | 世界観 | 有用性 | 安全性 | 合計 | 判定 |
|------|----------|--------|--------|--------|------|------|

### 改善提案
- [具体的な改善案を skill-improvement-log 形式で記述]
```

### 4. 前回比較
- `docs/eval-results/` ディレクトリ内の最新の結果ファイルと比較
- スコアが下がったskillには ⬇️、上がったskillには ⬆️ を付記
- 前回結果がなければ「初回評価」と記載

### 5. 結果の保存
- 結果を `docs/eval-results/YYYY-MM-DD.md` に保存
- 改善が必要なskillがあれば `docs/skill-improvement-log.md` に追記提案

## 注意事項
- 評価は現時点のリポジトリ状態に基づく客観評価
- 甘い採点は無意味。改善のために正直に採点する
- 全skillがA評価なら、評価基準自体を見直す必要がある
