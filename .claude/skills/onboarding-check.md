# Onboarding Check — 初回体験フロー検証

Product Unit の UX Researcher として、新規ユーザーが初めてアプリに触れる体験の完成度を検証してください。

## 手順

1. オンボーディング関連ファイルを特定:
   - `app/(culcept)/` 配下のトップページ・ホーム
   - `app/api/onboarding/` 関連エンドポイント
   - `lib/onboarding/` のロジック
   - Stargazer の初回観測フロー（`app/(culcept)/stargazer/`）
   - ログイン・サインアップ画面

2. 以下の観点で各画面を検証:

### 初回体験チェック項目

**フロー完全性**
- [ ] 未ログイン → サインアップ → 初回画面 の導線が途切れていないか
- [ ] 初回ユーザーに「次に何をすべきか」が明確か
- [ ] Stargazer 初回観測への誘導が自然か
- [ ] 各ステップに戻る手段があるか

**空状態（Empty State）**
- [ ] データがない状態で各画面は適切に表示されるか
- [ ] 空状態のメッセージが行動を促す内容か
- [ ] ローディング中の表示は適切か

**離脱リスク**
- [ ] 入力が多すぎるステップはないか（1画面3問以下が理想）
- [ ] 「なぜこれを聞くのか」の説明があるか
- [ ] スキップ可能な項目にスキップ導線があるか
- [ ] エラー時にユーザーが詰まらないか

**世界観の一貫性**
- [ ] 初回接触で Aneurasync の価値が伝わるか
- [ ] 「自己発見」の体験が最初から感じられるか
- [ ] トーンが知的・内省的・温かいか

3. 以下のフォーマットで報告

## 出力フォーマット

```
## オンボーディング検証 [日付]

### 総合評価: A / B / C / D
（A: 初回体験として完成 / B: 概ね良好、軽微な改善あり / C: 主要な問題あり / D: フローが未完成）

### フロー全体像
[ステップ1] → [ステップ2] → ... → [ホーム到達]
※途切れている箇所があれば :x: で明示

### チェック結果
| カテゴリ | 結果 | 詳細 |
|----------|------|------|
| フロー完全性 | :white_check_mark: / :warning: / :x: | ... |
| 空状態 | :white_check_mark: / :warning: / :x: | ... |
| 離脱リスク | :white_check_mark: / :warning: / :x: | 高リスク箇所: [N] |
| 世界観 | :white_check_mark: / :warning: / :x: | ... |

### 問題箇所と改善案
| 優先度 | 箇所 | 問題 | 改善案 |
|--------|------|------|--------|
| P0 | [file:line] | ... | ... |
| P1 | [file:line] | ... | ... |

### 初回ユーザーの想定体験シナリオ
1. [ユーザーが最初に見る画面の説明]
2. [次のアクション]
3. ...
→ 「自分って、そういう人間だったのか」に至るまでの最短パス: [ステップ数]
```
