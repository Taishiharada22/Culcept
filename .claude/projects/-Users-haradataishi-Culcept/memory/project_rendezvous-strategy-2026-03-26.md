Rendezvousは既存マッチングアプリの土俵で戦わない。Aneurasyncにしか作れない「観測→関係→言語化→再観測」のループ価値で勝つ。

**Why:** 全軸で既存アプリ（Pairs 800万MAU等）に劣位。アバター先行・片想い非表示・遅延報酬は従来型マッチングの土俵では弱み。しかし「自己理解ツールの延長としての人間関係マッチング」は誰も参入していない。

**How to apply:**
- 初期ローンチは friendship カテゴリ優先。既知の友人同士の関係性分析から開始
- romantic は MAU 1,000超えてから開放（候補0件問題の回避）
- 課金設計は「深度課金」（行動量ではなく自己理解の深さに課金）
- 価格・継続率・課金理由・市場規模は全て仮説扱い。検証対象として管理
- Genome Card を拡散の起爆剤に位置づけ
- AIカウンセラー層が構築済み（2026-03-26承認、DBマイグレーション適用待ち）

**次フェーズ優先事項:**
1. DB適用（counselor 6テーブル + RPC + CHECK拡張）
2. friendship体験の完成度向上
3. Genome Card連動の拡散設計
4. 自己理解課金の成立性検証
