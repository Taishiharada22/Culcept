## 2. State A = ① 候補が出る（選び始める）
**画面構造（縦）**
```
おすすめの候補                               ← 見出し 14px bold
┌─────────────────────────── card (W, 高さ ~232px) ──┐
│ [72×72 thumb]  名前 16/2行         [相性高め]badge │  ← 上段: サムネ左 + 名前右 + 相性右上
│                理由文 13/2行                        │
│ 🚶約5分(目安)  🏷️カフェ  💜相性                      │  ← 主要属性チップ（honest のみ）
│ 📍 東京都江東区平野1-4-8 …                           │  ← 住所 11.5
│ [ 詳細を見る  outline 50% ][ 比較する ⇧ filled 50% ]│  ← 比較導線
└────────────────────────────────────────────────┘
   └ 次候補カードの上端が ~40px のぞく（1主役 + 次少し）
```
- **主役要素**: いま見ている **1 候補カード**。
- **情報階層**: 名前 > 相性 > 理由 > 主要属性 > 住所 > アクション。
- **カードサイズ**: W×~232px。サムネ 72px。**小さすぎ禁止**＝文字を詰めない・行間 1.5。
- **ボタン配置**: カード下端に 2 ボタン横並び（各 ~50%、44px）。
- **余白**: カード間 12px、内側 16px。
- **状態遷移**: 縦 snap（`snap-mandatory`）で 1 候補ずつ送る。下に次候補が少し見える高さ（container ≈ card+40px peek）。
- **現実装との差分**: 現状はカードが小さく文字が詰まり、情報階層が崩れている。→ サムネ拡大・理由を読めるサイズ・チップを整理・1主役+のぞきを明確化。

**画面構造**: ①の **同じカードが同じ位置で縦に拡張**（accordion）。周囲（フォーム・他候補）はそのまま、当該カードだけ大きくなる。
```
┌─────────── card 展開 (W, 高さ ~452px) ───────────┐
│ 名前 17                                   [∧ 畳む]│
│ カフェ ·  [相性高め]                                │
│ ┌──── abstract media banner  W×150 ────┐          │  ← 地図/写真の位置（honest: abstract）
│ └──────────────────────────────────────┘          │
│ 📍 東京都江東区平野1-4-8 清澄白河フラッグA棟         │
│ 🚶約5分(目安)[計算]  🏷️カフェ[事実]  💜相性          │  ← 属性 + evidence
│ ┌ なぜここを選ぶ？ ──────────────────────┐         │
│ │ 徒歩約5分(目安)で、会議前に余白を…(よく行く)│         │
│ └────────────────────────────────────────┘         │
│ [ 閉じる ‹ 50% ][ 比較する ⇧ 50% ]                  │
└────────────────────────────────────────────────┘
```
- **主役要素**: 展開した候補の **詳細（メディア＋なぜ＋属性）**。
- **情報階層**: 名前 > メディア > 住所 > 属性(evidence) > なぜここを選ぶ？ > アクション。
- **カードサイズ**: W×~452px（①の ~232 から **その場で拡張**）。メディア W×150。
- **ボタン配置**: 下端に 閉じる / 比較する。
- **余白**: なぜブロック内 16px、メディア上下 12px。
- **状態遷移**: ①の「詳細を見る」or カードタップ → 当該カードが拡張（高さアニメ）。∧/閉じるで①へ収縮。**別サーフェスは出さない**。
- **現実装との差分**: REDO-4 は下から出る別ボトムシート＝**誤り**。→ **同領域内のインライン詳細展開**に戻す（REDO-3 の方向が正しかった）。

## 4. State C = ③ 候補を比較する（比較専用の大きな状態 / ★全画面ページにしない）
**画面構造**: 比較**専用サーフェス**（大きめの比較モーダル／sheet・角丸・周囲を暗転）。アプリ全画面ページ化はしない。主役は**比較表**。
```
┌──────── 比較サーフェス (大きめモーダル) ────────┐
│ ‹ 戻る      候補を比較          会議前 / 2件     │  ← ヘッダー
│ ┌ 候補A ────────┐ ┌ 候補B ────────┐ ▲▼        │  ← 2候補ヘッダー(サムネ+名前) + 入替
│ │[mini] 名前      │ │[mini] 名前     │           │
│ └────────────────┘ └────────────────┘           │
│ ── 候補を比較 ────────────────────────────────  │
│ 🚶 徒歩    │ ✓約5分(目安)紫 │ 約15分(目安)        │  ← 比較表(主役)・優位セル紫+✓
│ 💜 相性    │ よく行く…      │ —                  │
│ 🏷️ 種別    │ カフェ         │ カフェ              │
│ 📍 住所    │ 江東区…        │ 渋谷区…             │
│ ── ✨ おすすめ ───────────────────────────────  │  ← おすすめサマリー
│ ブルーボトル が、徒歩の点で合いそうです            │
│ [   この場所にする（選択時のみ・下部固定）   ]    │  ← CTA
└────────────────────────────────────────────────┘
```
- **主役要素**: **比較表**（2 候補 × 目的レンズの軸）。
- **情報階層**: 2候補ヘッダー > 比較表 > おすすめサマリー > CTA。
- **寸法**: ヘッダー 56px、候補ヘッダー 88px、表 行高 52px、おすすめ 72px、CTA 56px。サーフェスは内容に応じ ~70–85vh の比較モーダル（full-bleed のアプリページにしない）。
- **ボタン配置**: 上＝戻る、右＝目的/件数、下＝確定 CTA（候補タップで選択→確定）。
- **余白**: 行 padding 縦12/横14、ブロック間 16。
- **状態遷移**: ①/②の「比較する」→ 比較サーフェスが前面に。戻るで①へ。右候補は ▲▼/スワイプで入替。
- **現実装との差分**: REDO-4 は `fixed inset-0` の**アプリ全画面ページ**＝CEO 意図とズレ。→ **比較専用モーダル（大きめ・角丸・暗転）**に変更。主役は比較表。

## 8. REDO-6 設計デルタ（REDO-5 実機監査 → CEO 差戻し・コード書かず設計のみ提出）
REDO-5（①密度/②accordion/③84vh sheet）を CEO が「改善は見えるが理想とは別物・①②③すべてズレ」と判定。3 状態を再設計（実装は本デルタの GO 後）。

### ① 候補が出る（★1候補のみ＋上下スワイプ＝「捲る」体験）
- **現状どこが違うか**
  1. スクロールしなくても次候補がのぞいて見え、**一覧感**が残る（peek を出していた）。
  2. 「1候補が画面を占有する主役」になっておらず、カードが小さい。
  3. 送り方が「スクロール」で、CEO 意図の「上下スワイプで捲る」になっていない。
- **どう直すか**
  1. 候補領域を**1カード固定窓**にし、**常に1候補だけ**表示（peek 廃止＝次カードを見せない）。
  2. **上下スワイプ／▲▼で次・前へ送る pager**（snap 1 枚送り・`1/3` カウンタ＋ドット）。一覧 DOM をやめ「現在の1枚」だけ前面に。
  3. カードを窓いっぱいの主役サイズにし、サムネ/名前/相性/理由/属性/住所/簡易map/2導線を読めるサイズで配置。
- **画像のどこを再現するか**: add-search1 の「主役候補が大きく1枚」「候補理解に必要な情報が揃う」構造（ただし“次候補が並ぶ一覧感”は出さない＝捲る UX に置換）。

### ② タップで詳細がひらく（★同じカードがその場で大きく詳細化する明確な別状態）
- **現状どこが違うか**
  1. 展開が弱く「②がほぼ存在しない」ように見える（同領域の小さな差分にしか見えない）。
  2. ①→②の**連続性**（同じカードが大きくなった、という一目の繋がり）が弱い。
  3. 画像/地図/理由/属性が**主役**として大きく出ていない。
- **どう直すか**
  1. ①の1カードをタップ→**同じカードが同じ位置で大きく morph**（高さアニメ・shared element 的連続性）。別カード/別シートにしない。
  2. 展開時は**大メディア（place tile＋簡易map を横長 banner で大きく）**を最上部の主役に、続けて住所・属性(evidence)・「なぜここを選ぶ？」を階層表示。
  3. ①と②で**ヘッダー（名前＋相性）位置を共有**し「同じカードが開いた」と分かる遷移にする（∧で①へ収縮）。
- **画像のどこを再現するか**: add-search2 の「同じカードが大きく開き、写真/地図/理由/属性が主役」になった詳細状態（∧で畳む）。

### ③ 候補を比較する（★比較が主役の専用モード・密度と説得力を上げる）
- **現状どこが違うか**
  1. 比較表の**情報密度が薄い**（行が少なく見え、主役に見えない）。
  2. 優位の**説得力が弱い**（薄紫＋✓だけで差が一目で響かない）。
  3. 「上に2候補・下に比較表で全体が比較主役」という**構図の強さ**が足りない。
- **どう直すか**
  1. 2候補ヘッダーは**コンパクト**にして上に固定、**比較表を画面の主役**（行高拡大・ゼブラ・項目アイコン＋ラベルを左に固定列）に。
  2. 優位を**強い視覚差**で（勝ち列セルを塗り＋✓＋「近い/余裕」等の短い優位ラベル・負け側は淡色）。honesty 維持（表示値に差がある行のみ）。
  3. 未確認は密度のため**畳んだ小セクション**で正直に明示。最下部に**✨おすすめ（根拠句）＋確定 CTA**を強く。70–85vh の専用ボトムシート（背景暗転）は踏襲。
- **画像のどこを再現するか**: add-search3 の「上＝2候補ヘッダー／中＝比較表（主役・行ごと優位ハイライト）／下＝おすすめ」の構図と密度。

### honesty（不変・全状態共通）
写真→category タイル / 地図→簡易 map タイル（装飾・実地図でない） / Wi-Fi・電源・静か・雰囲気・営業時間→捏造せず「未確認」 / 優位は表示値差のみ / B は約・目安。**密度は寄せるが捏造はしない**。

## 9. REDO-7 設計（★最重要: 既存「右枠内」発想の禁止・左レーン侵入 / 実装前 GO 待ち）

### 9.0 構造把握（重要な前提）
予定追加 UI は `AddAnchorModal`(`GlassModal size=md`=`max-w-lg`=**512px**)。画面中央に `left-1/2 -translate-x-1/2` で fixed。背景は `bg-slate-900/20 backdrop-blur-sm` ＝**左の月カレンダー／タイムラインがうっすら透けて見える**。CEO の「右枠内禁止／中央を跨いで左レーンに侵入」＝**この modal の幅を 512px の枠に固定するな、候補 UI 表示時はもっと幅を取り、画面中央軸より左へ広がって左カレンダーの上に被さって浮かぶ主役にしろ**、の意味。①②は modal の `size=md` 制約を脱出する必要がある。③は既に portal 全画面下シートで脱出済。

### 9.2 ① 候補が出る（再設計）
- **表示要素一覧（再整理）**:
  1. 見出し「おすすめの候補」＋ `i/N`
  2. 1 候補のみ（次候補非表示）
  3. 候補名（2行まで）
  4. 相性バッジ
  5. 短い理由文（1〜2行）
  6. **メディア＝写真相当タイル**（abstract place tile・~ 200×140px）
  7. **小 map tile**（~ 120×100px）
  8. 主要チップ 3〜4 個（徒歩／種別／（評価可能なら）Wi-Fi・電源・静か等）
  9. 営業時間相当（持つなら値、無ければ「未確認」を 1 チップで）
  10. **位置情報＝1 行 ellipsis**（フル住所はツールチップ／②で全文）
  11. CTA: 「詳細を見る」「比較に追加」
- **開始位置**: 「どこで？」入力欄の下＋「おすすめの候補」見出し直下。modal 内ではなく、modal の境界を跨いで描画（portal overlay・modal の上の z）。
- **横幅**: `W_LENS` = `min(880px, 96vw)`。既存 modal(512px)より大幅広い。
- **左レーン侵入量**: 中央軸から **左へ −230px / 右へ +210px**。左カレンダー（月セル＋タイムライン左カラム）の上に半被り。lens 本体は不透明白カード／背景に薄い暗転（modal そのままは閉じない、操作競合は z で解決）。
- **省略する情報**: **位置情報＝1 行 ellipsis**（最大 ~28 全角）／長い名前は 2 行で打ち切り／理由は 2 行。
- **遷移**: 上下スワイプ／▲▼／ドットで前後候補へ「捲る」。**ハナから 1 件のみ render（peek なし）**。

### 9.3 ② タップで詳細がひらく（再設計）
- **表示要素一覧**:
  1. 候補名（行送り）
  2. 相性バッジ
  3. ∧ 折りたたみ
