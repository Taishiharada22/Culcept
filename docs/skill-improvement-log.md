## 記録フォーマット

各エントリは以下の形式で記録する:

```
### [YYYY-MM-DD] /skill-name — 問題の要約

- **発見経緯**: eval / CEO修正 / 実運用中の気づき
- **失敗内容**: 何が起きたか、何が足りなかったか
- **CEO修正**: CEOがどう直したか（あれば）
- **根本原因**: なぜskillがそう振る舞ったか
- **改善案**: skill定義のどこを変えるか
- **ステータス**: 🔴 未対応 / 🟡 改善中 / 🟢 完了
- **次回見直し**: YYYY-MM-DD（次の weekly review で確認）
```
