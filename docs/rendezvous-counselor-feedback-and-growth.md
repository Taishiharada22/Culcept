## 1. フィードバックシステム：3層設計

### 1.1 なぜフィードバックが結婚相談所の最大の弱点か

リサーチ結果:
- 断り理由は「フィーリングが合いませんでした」に均一化される
- カウンセラーは会員の「お断り理由の統計グラフ」を見れるが、会員には見せない相談所が多い
- 改善可能な理由（身だしなみ、会話スキル）と改善不可能な理由（容姿、年齢）が区別されない
- フィードバックは自由テキスト・口頭ベース。定量分析なし
- パートナーエージェントのPDCAが業界最先端だが、まだ人力依存

### 1.2 第1層：行動観測フィードバック（本人が気づいていない自分）

ユーザーが「言う」のではなく、Counselorが「観測する」:

**チャット中のシグナル:**
- 返信速度の変化（話題ごとの加速/減速）
- 話題への食いつき度（質問の深さ、追加質問の有無）
- 感情表現の変化パターン（絵文字使用、文体の変化）
- 主導権の取り方（質問する側か答える側か）

**電話中のシグナル（将来実装）:**
- 沈黙の長さ・頻度
- 笑いの自然さ
- 話題転換のタイミング
- 声のトーンの変化

**パターン横断分析:**
- 「3人目以降で話題提供頻度が下がる」
- 「○○の話題で返信が加速する」
- 「初回は質問が多いが、3回目以降は受動的になる」

既存相談所では**絶対に不可能**（カウンセラーは会話に同席しない）。

### 1.3 第2層：自己発見フィードバック（関係性マインドフルネス）

既存: 「どうでしたか？」→「楽しかったです」（解像度ゼロ）

**Rendezvous Counselorの問い:**
```
「今日の会話で、一番自然に言葉が出た瞬間はどこだった？」
「逆に、少し考えてから返した場面はあった？」
「相手の返事で、意外だったものはある？」
「帰り道、身体は軽かった？重かった？」
「会話中、身体のどこかに変化を感じた瞬間はあった？」
「相手が笑ったとき、あなたの中で何が起きた？」
```

回答そのものより、回答に至る反応パターンが観測データ。これは内受容感覚の訓練であり、使い続けるほど「なんとなく」の解像度が上がる。

### 1.4 核心：第1層と第2層の照合

**ユーザーの自己報告と行動観測データのズレが最も価値の高い観測データ。**

例: 「楽しかった」と報告しているが、チャット中の返信速度は後半で明らかに遅くなっていた。

Counselorの対応: 攻撃的に指摘するのではなく、
```
「前半と後半で少し雰囲気が変わっていたけど、何か感じた？」
```
とユーザー自身に気づかせる。

この設計は**関係性マインドフルネスのトレーニング**。AIが全てを判断するのではなく、ユーザー自身の気づきの力を育てる。結婚相談所は「カウンセラーに依存する構造」。マッチングアプリは「自分で全部やる構造」。Aneurasyncは**「自分で気づく力をAIが育てる構造」**。第三のカテゴリ。

### 1.5 第3層：相互フィードバック（Exchange Protocol経由）

双方のCounselorが行動観測データに基づいて温度感を判定。既存相談所の「本人→自分のカウンセラー→相手のカウンセラー」の2回変換ロスが最小化。

### 1.6 累積分析：「自分のパターンが見える」

10回のチャット・5回の通話・3回の対面を経た後:
```
「あなたは最初の会話で相手に質問を多くする人と、長く続く傾向がある」
「声のトーンが穏やかな相手を選ぶとき、交際の満足度が高い」
「条件面を早い段階で確認する相手とは、2回目以降に温度が下がりやすい」
「あなたが『楽しかった』と報告した相手と、実際にリラックスしていた相手は、3割ズレている」
```

最後の一行が核心: **自己認識と実際の行動のズレを可視化する。** IBJのAIマッチングも自己申告の条件と行動履歴のパターンマッチングであって、「本人の自己認識のズレ」には踏み込んでいない。

## 2. コールドスタート解決：パラダイムシフト

### 2.1 問題の本質

マッチングサービスの宿命は二面市場のコールドスタート。IBJに頼るのは「違う気がする」（CEO判断）。正しい。IBJは「結婚したい人の集合」。Aneurasyncは「自分を知りたい人の集合→自然に関係性が生まれる場」。入口が違う。

**Aneurasyncは二面市場ではない。** 全ユーザーが同じ側にいるネットワーク。恋愛も友達もパートナーも同じ母集団から生まれる。

### 2.2 解法：「知らない人を見つける」から「知っている人を理解する」へ

**Rendezvousの起動点を、新しい出会いではなく、既存の関係の理解にする。**

```
従来: Stargazer（自己理解）→ Rendezvous（知らない人と出会う）
転換: Stargazer（自己理解）→ Rendezvous（知っている人との関係を理解する）
      → 関係性マップが広がる → 新しい出会いが自然に生まれる
```

**なぜ2人いれば機能するか:**
- 招待行為自体がプロダクトの一部
- 既知の関係で精度を体感 → 未知の人への推薦を信頼できる
- 恋愛だけでなく友情・家族・同僚 → ターゲットが全人口

### 2.3 ネットワーク科学的根拠

Aneurasyncでは**エッジ（関係性）自体がデータ**。AとBの関係を観測すると、個人の理解が深まるだけでなく「AのようなタイプとBのようなタイプの組み合わせでは何が起きるか」という関係性パターンデータが得られる。ユーザーが増えるほど、マッチングの根拠が指数関数的に豊かになる。

**既存の全てのマッチングサービスはノード（個人プロフィール）で判断。Aneurasyncはエッジ（関係性の実データ）で判断できる。** これは根本的に異なる情報の質。

### 2.4 他業界のアナロジー

**Instagram:** 写真フィルターアプリ（単独価値）→ 共有したい → ネットワーク効果。プロダクト自体の単独価値が成長の燃料。

**Spotify:** 曲を聴く → Discover Weekly（自分の知らない自分の好み）→ プレイリスト共有。自分を知る → 関係性を理解する → 新しい出会い、の同構造。

**Facebook:** ハーバードの「既に知っている人のつながりのデジタル化」から始まった。Aneurasyncも「既に知っている人との関係の深い理解」から始める。

**Tesla:** Roadster（少数・コンセプト証明）→ Model S（プレミアム）→ Model 3（マス）。最初から大衆を狙わず小さく深く刺してから広げる。

### 2.5 Genome Cardの成長エンジン機能

Genome Cardは**Aneurasyncの「写真フィルター」（Instagram的成長エンジン）**になりうる:

```
1. Stargazerで自己理解 → 美しいGenome Cardが生成される
2. 「自分って、こういう人間だったのか」→ 感動 → 共有したくなる
3. 友人がGenome Cardを見る → 「自分のも欲しい」→ ダウンロード
4. 友人がStargazerを始める → 新しいGenome Card → 共有…
```

**CEOの洞察:** 小中高校生の間では、名前占いや星座相性診断が爆発的に使われている。Genome Card交換＝相手との相性がわかる＝価値がある、はこの文化の完全上位互換。「ちゃっちいものではなく、完璧なロジックで動いている」から競合がいない。

## 3. 招待トークン設計

### 3.1 基本思想

**招待＝相手のためではなく、招待＝自分自身のため。** 人間はそれぞれが主人公。

- AがBを招待しても、AはBとの相性を見る義務はない
- 招待によりポイントが付与される
- AはそのポイントをBではなくCとの相性を見るために使ってもいい
- ポイントの使い方は招待者・被招待者の自由

### 3.2 ポイント付与体系

```
招待した相手が会員登録完了: 25pt
招待した相手がPhase 1到達:  25pt
招待した相手がPhase 2到達:  50pt
────────────────────────
合計:                       100pt
```

**招待送信上限: 月5人**（雑な大量招待を防ぐ。ポイント獲得上限はなし——相手のPhase到達次第で自然に制限される）

### 3.3 トークン変換レート

```
100pt = Friendship相性トークン 1枚（既知ペアの関係性インサイト1回分）
200pt = Discovery preview 1枚（まだ出会っていない人の相性プレビュー1回分）
```

**Friendshipトークンの用途:** Free枠で1組は無料だが、2組目以降はトークンまたはCore課金が必要。

**Discovery previewの用途:** Rendezvous Romance/Partner開始前に、候補の相性概要を覗ける。本接続にはRomance/Partner課金が必要。

### 3.4 トークンの命名（仮）

世界観に合わせた命名はCEOの判断に委ねるが、機能的な仮称:
- Resonance Token（共鳴トークン）= Friendship相性
- Discovery Token（発見トークン）= 新しい出会いのプレビュー

## 4. 成長ステージ設計

### 4.1 全体像

```
Stage 0: Stargazer単体の価値（会員数は関係ない）
├── 自己理解ツールとして（MBTI/エニアグラムの完全上位互換）
├── Genome Cardの共有で自然拡散
├── Alter基本会話
└── KPI: ユーザー数、Phase 2到達率、Genome Card共有率

Stage 1: 既知ペアの関係理解（2人いれば機能する）
├── 招待トークンで既知ペアのインサイト
├── Friendship（同年代の友人関係理解）
├── 精度の体感 → 信頼の構築
└── KPI: 招待ループ率、ペア数、インサイト評価スコア

Stage 2: 新しい出会いの解放（母集団が臨界点）
├── Romance開始（友達カテゴリの次）
├── Counselor 1対1テキスト接続
├── 既知ペアで証明された精度が信頼を担保
└── KPI: マッチング成功率、関係継続率

Stage 3: Partner本格稼働（成功実績後）
├── 書類認証、成婚プロセス
├── Counselor Exchange Protocol全機能
├── 成功事例の蓄積 → 信頼性の証明
└── KPI: 6指標体系（後述）
```

### 4.2 Partnerの起動条件

**課金開始タイミング:** Partner登録（無料）→ 待機 → **その人にとってのマッチング候補がN人以上揃った時点で課金開始通知**

- 全体の人数ではなく、その人にとっての母集団で判断
- 「紹介できる相手がいないのに課金される」不信感がゼロ
- MVP段階ではシンプルな人数閾値（例: Partner登録50名到達で全員に開始通知）でも可

## 5. カテゴリ別設計

### 5.1 Friendship

**基本方針:** 無料で提供（コールドスタート解消の起点）

**無料/有料の境界:**
```
Free: 既知ペア1組（肯定全文 + 否定ティーザー「ここに気づきがあります」）
Core: 既知ペア無制限 + 全インサイト（肯定＋否定＋改善提案）+ 累積分析 + 履歴比較
```

**恋愛目的の侵入防止（性別制限ではなく行動制限）:**

性別で制限すると、LGBTQ排除になり、異性間友情の理解も失われる。代わりに行動観測で検出:

```
1回目検出: Counselorが「この接続はFriendship目的です」と本人にリマインド
2回目検出: 相手側のCounselorにも通知
3回目検出: 接続を凍結し、Romance/Partnerへの移行を案内
```

検出シグナル: 過度な外見言及、デートの誘い、性的な話題、1対1で会うことへの執着

**サブカテゴリ:**
```
friend: 1対1の友人関係理解（Free枠あり）
community: 共通の関心軸でのグループ（Core以上、Phase 2以上で参加可能）
business: 「仕事での相性」に特化したインサイト（Core以上）
```

community/businessはFriendshipの下位機能ではなく拡張機能。Core以上で解放。

### 5.2 Romance

**月額12,800円（Core全機能込み）**

- 新しい出会い（まだ知らない人との接続）
- Counselor基本版
- テキスト接続
- Partnerへの導線（「この人との関係を結婚視野で考えたい」→ Partner移行案内）

### 5.3 Partner

**月額24,800円 + 成婚料330,000円**

- Core全機能 + 結婚マッチング
- Counselor完全版（Exchange Protocol付き）
- テキスト → 電話 → 対面の段階的接続
- 書類認証（独身証明・収入証明等）

**Partner Standalone:** Romance機能は含まない。「結婚相談所利用者」は遊びの恋愛を求めていない。混在は信頼を損なう。

**Partner + Romance（月29,800円 + 成婚料）:** 「結婚も視野に入れつつ、まず恋愛から」という人向けのオプション。

### 5.4 Premium

**月額39,800円 + 成婚料330,000円**

- 全機能
- Counselor優先（応答品質・分析深度の最大化）
- 専用サポート
- 対面推薦の優先枠

## 6. 未成年向け設計

### 6.1 年齢区分

| 年齢 | 利用可能 | 条件 |
|------|---------|------|
| 12歳以下 | 利用不可 | — |
| 13-15歳（中学生相当） | Stargazer / Origin / Calendar / Genome Card / 既知ペア相性 | 親権者の同意必須 |
| 16-17歳（高校生相当） | 同上 | 親権者の同意必須 |
| 18歳以上 | 全機能 | 通常の利用規約 |

### 6.2 開放する機能

- **Stargazer:** 自己概念の発達が本格化する13歳から意味を持つ（ピアジェの形式的操作段階）
- **Origin:** 日記・行動パターン
- **Calendar:** 天気×気分×コーデ提案
- **Genome Card:** 交換 + 招待トークン取得 + 選んだ相手との相性インサイト（成年と同等）

### 6.3 禁止する機能

- **Rendezvous全カテゴリ（Friendship含む）の「新しい出会い」:** 出会い系サイト規制法の対象回避
- **ただし、既知ペアの関係性インサイトは許可:** 既に知っている人との理解を深めるのはマッチングではない

### 6.4 課金設計

```
13-17歳: 基本無料
  - Stargazer / Origin / Calendar / Genome Card: 無料
  - 既知ペア相性インサイト: 招待トークンで利用（一定数を無料付与）
  - Core課金: 親権者の同意＋親権者のクレジットカード必須

18歳以上: 通常の料金体系
```

**戦略的理由:** 中高生を無料にするのは10年後の市場を作る投資。中高校生の間でGenome Card交換が流行 → 大学進学後にCore/Romance/Partnerへ自然に移行。若年期からAlterのデータが蓄積 → 20代でRendezvousに入る時点でAlterの精度が桁違いに高い。

### 6.5 未成年データの安全設計（最優先）

```
- 未成年データは成年データとは別のストレージポリシー
- 18歳到達時に本人確認 → データ継続利用の明示的同意を取り直す
- 親権者がいつでもデータ削除を要求できる
- 未成年の観測データはRendezvousマッチングのアルゴリズム学習に使用しない
- プロファイリング結果の第三者提供は絶対禁止
- 個人情報保護法: 16歳未満は親権者同意が必須（ガイドライン準拠）
- Apple/Googleストアポリシー: 13歳未満はCOPPA準拠が必要
```

## 7. 成果指標：6指標体系

### 7.1 設計思想

既存相談所の「成婚率」は定義が各社で異なり信頼性が低い（同じ会社でも計算式次第で21.7%～61.4%に変動）。Aneurasyncは「進行」だけでなく「品質」も測定し、透明に公開する。

### 7.2 進行指標（3つ）

| 指標 | 定義 |
|------|------|
| **Connection Progress Rate** | 初回テキスト接続 → 継続対話 → 対面まで進んだ割合 |
| **Commitment Rate** | 交際専念・婚約意向確認まで進んだ割合 |
| **Marriage Rate** | 一定期間内に婚姻・婚約・同居合意に至った割合 |

### 7.3 品質指標（3つ）— Aneurasync独自

| 指標 | 定義 | なぜ革新的か |
|------|------|------------|
| **Relationship Confidence Score** | 交際/成婚時点での双方の確信度（Counselor観測ベース） | 「とりあえず成婚」ではなく「確信を持って成婚」を測れる |
| **Honest Exit Rate** | Counselorが「進めない方がいい」と判断し、ユーザーが納得して撤退した割合 | **既存相談所が絶対に出せない指標。信頼性の証明** |
| **Self-Understanding Delta** | Rendezvous開始前→後での自己理解スコアの変化 | 「出会えたか」だけでなく「自分を知れたか」を測る |

### 7.4 分母の定義

**「Partner入場後、最低1回のテキスト接続を行ったユーザー」**

- 「登録だけして放置」は除外
- 1回接続して合わなかった人は含む
- 厳しすぎず甘すぎない
- 計算方法を全て公開し、業界の不透明性と差別化

### 7.5 Honest Exit Rateが最大の武器

「うちはこれだけの割合でユーザーに正直に撤退を勧め、ユーザーもそれに納得しました」——これは:
- 既存相談所の「成婚バイアス」との決定的な差
- ユーザーへの「あなたの利益を最優先にしている」というメッセージ
- メディアや口コミで最も引用されやすい指標（「こんな指標を公開する相談所は初めて」）
