## 1. テストユーザー招待条件

### 必須条件
- Stargazer 観測を 30 問以上完了済み（性格データが十分に蓄積されている）
- Rendezvous オンボーディング完了済み（プロフィール・カテゴリ設定済み）
- 信頼できる知人・関係者（CEO が直接招待）

### 推奨条件
- 25〜40 歳（Partner ターゲット層）
- 結婚意向がある、またはそのシミュレーションに協力可能
- テスト後のフィードバック提供に同意

### 招待フロー
1. CEO がユーザーに直接連絡
2. アカウント作成 → Stargazer 観測 → Rendezvous オンボーディング
3. `enabled_categories` に `"partner"` を含むようプロフィールを設定
4. Life Plan 35 問に回答
5. 身元確認（L3）の提出を依頼

### 初期テスト規模
- **Phase 1**: 4〜6 名（2〜3 ペア分）
- **Phase 2**: 10〜20 名（フィードバック反映後）

## 2. 審査運用マニュアル

### 管理画面アクセス
- URL: `/admin/rendezvous/verifications`
- 権限: `isAdminEmail()` に登録済みのメールアドレスのみ

### 審査フロー

```
管理画面にアクセス
  → 「確認待ち」タブで pending 件数を確認
  → 各件の書類・セルフィーを目視確認
  → 判断:
     ├─ 問題なし → 「承認」
     ├─ 不鮮明・不一致 → 「再撮影依頼」(理由を記入)
     ├─ 明確な詐称 → 「リジェクト」(理由を記入)
     └─ 安全上の懸念 → 「凍結」(理由を必須記入)
```

### 審査判断基準

| 確認項目 | 承認条件 | 却下条件 |
|----------|----------|----------|
| 身分証の種別 | 運転免許証 / パスポート / マイナンバーカード | 上記以外の書類 |
| 身分証の鮮明さ | 文字・写真が判読可能 | ぼやけて判読不能 |
| 身分証の有効期限 | 有効期限内 | 期限切れ |
| 本人とセルフィーの一致 | 同一人物と判断可能 | 明らかに別人 |
| 年齢 | 18 歳以上 | 18 歳未満 → **凍結(fraud:underage)** |

### 審査 SLA
- **目標**: 提出から 24 時間以内に審査完了
- **β期間**: CEO が 1 日 2 回（朝・夜）に審査キューを確認

### 審査後の状態変化

| アクション | verification_status | review_status | verification_level |
|-----------|--------------------|--------------|--------------------|
| 承認 | `verified` | `approved` | 3 |
| 却下 | `rejected` | `rejected` | 変化なし |
| 再撮影依頼 | `rejected` | `rejected` | 変化なし |

## 3. 凍結 / 再申請フロー

### 凍結トリガー（β期間中）

| トリガー | frozen_reason | 対応 |
|----------|--------------|------|
| なりすまし疑い | `report:impersonation` | 管理者が凍結 → 調査 → 解除 or 永久凍結 |
| 身分証詐称 | `fraud:document` | 即時凍結 → CEO 判断で永久凍結 |
| 年齢詐称（18歳未満） | `fraud:underage` | **即時永久凍結** |
| 管理者裁量 | `admin:discretion` | note に詳細記載 |

### 凍結ユーザーへの影響
- 候補一覧から完全除外（他ユーザーから見えなくなる）
- いいね / チャット / 面会すべて不可
- 画面に「アカウントが一時停止されました」表示

### 凍結解除
- 管理画面で「凍結解除」ボタン
- `fraud:document` / `fraud:underage` は **CEO 承認なしに解除不可**
- 解除後、全機能が即時復帰

### 再申請フロー
1. ユーザー画面に却下理由が表示される
2. 「再提出」ボタンで身分証・セルフィーを差し替え
3. `verification_status=pending`, `review_status=pending` に戻る
4. 審査キューに再度入る
5. 再申請回数の上限なし（β期間中。悪用が見られれば制限追加）

## 4. 問い合わせ対応方針

### β期間中の対応チャネル
- **主**: CEO が直接対応（知人ベースのため）
- **副**: アプリ内「問い合わせ」→ メールで CEO に通知

### 想定される問い合わせと回答テンプレート

#### Q1: 本人確認はなぜ必要ですか？
> Partner は結婚を前提とした出会いの場です。お互いの安全と信頼のために、身分証による本人確認をお願いしています。確認後はより安心してマッチングをお楽しみいただけます。

#### Q2: 審査にどれくらいかかりますか？
> 通常 24 時間以内に完了します。お急ぎの場合はお知らせください。

#### Q3: 身分証が却下されました。どうすればいいですか？
> 却下理由をご確認のうえ、鮮明な画像で再提出をお願いします。不明な点があればお問い合わせください。

#### Q4: アカウントが凍結されました。
> 安全上の理由により一時停止されています。詳細については個別にご連絡いたします。

#### Q5: マッチした相手の本人確認レベルはわかりますか？
> 候補カードに確認バッジが表示されます。L3（本人確認済み）以上の方には「本人確認済み」バッジが付きます。

### エスカレーション基準
- 凍結の異議申し立て → CEO 判断
- 法的な問い合わせ → CEO 判断（弁護士相談が必要な場合あり）
- バグ報告 → Build Unit が対応
