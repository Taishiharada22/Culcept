## §0 一行要約

> 完璧な fit は独立軸の総和では作れない。状態は **dense**（9 族 / **102 構築子** / **634 指標**、内部は 50+ 族へ拡張可）に保ち、軸どうしの**相互作用（baggage×stairs×crowd の超加算、rain×outdoor×fallback の gate、静けさ×recoveryStyle の符号反転）**を**型付きレジストリ**で明示する。相互作用は**既存 component の修飾子であって新たな並列スコアにしない**（二重計上を構造で塞ぐ）。利用者への問診は compact、内部状態は dense ―― この非対称が「自分って、そういう人間だったのか」を支える。

## §1 天井の否定 ―― 18 は MVP 第一層であって上限ではない（密度の実証）

workflow が seed した第一層だけで:

| 観点 | T11-A3 着地 | 本設計（第一層の実測） | 内部到達目標 | 利用者向け |
|---|---|---|---|---|
| construct family | — | **9 族** | **50+ 族** | — |
| L1/L1b/L2 construct | 18 | **102 構築子** | 数百 | 問診は 5–9 問束 |
| observed indicator | ~90 | **634 指標**（H_route だけで 22 構築子/150+ 指標） | 数千 | 内部のみ蓄積・非表示可 |
| interaction term | 暗黙 | **明示レジストリ（§4）** | 段階拡張 | — |

- **非対称の原則**: 問診は compact（観測の入口）、内部 state は dense（観測した断片を高解像度で保持）。**露出 ≠ 蓄積**。可視性は L9 で制御（default private）。
- **成長は additive 4 手段のみ**（T11-A2 §10 と同一）: 新構築子 / 新指標 / 新 facet field / 新 context modifier。エンジン分岐ゼロ（全構築子が同一 `Observed<T>`/gate-first/provenance スキーマのインスタンス）。
- 各族の EXPAND ノートが「この族は 50+ 族へ育つ第一層」と明記（例: A_sensory→B_social/C_pace/…、H_route→leg 粒度細分）。

## §2 型付きレジストリ ―― `as-const`、`IndicatorKey = string` は最終形として却下

| 方針 | 内容 | 根拠 |
|---|---|---|
| 族/カテゴリ単位 `as-const` | 各 family・category を `as const` で定義し IndicatorKey を **literal union** として導出 | テスト・safety gate が key を型で固定 |
| 型ロック | safety-critical(L5 veto) / double-count 禁止ペア / valence 方向 を型レベルで参照可能に | コンパイル時に二重計上・veto 漏れ検出 |
| escape hatch | `{ known: KnownIndicatorMap; ext?: Record<ExtKey, IndicatorSpec> }`。ext は **confidence 既定低・safety-critical 不可（型で禁止）** | 数百指標への増殖を型崩壊なしに許容 |
| 却下 | `type IndicatorKey = string` を**最終形として却下**（テスト不能・veto 取違え・double-count 検出不能） | safety/test 前提を満たさない |

原則: known は型ロック / ext は緩いが安全境界（veto 不可・provenance 既定低）で囲う。

## §4 ★ 相互作用レジストリ（本書の中核・GPT を超える型化）

> 完璧な fit は独立軸の総和では作れない。状態×状態の非線形合成を**型付き `InteractionTerm`** で明示する。

### §4.1 型モデル（本書著者の設計・combiner 種別 + scope 分類 + confidence 連鎖 + 修飾子不変条件）

```ts
type InteractionScope =
  | "within_object"   // 同一 object の属性間（この宿の階段×荷物）
  | "cross_object"    // object 間（ホテルdrop × 観光順序）
  | "object_context"  // object × FitContext（雨 × 屋外 entity）
  | "object_user"     // object × user trait/state（静けさ × recoveryStyle）
  | "object_group";   // object × participant 群（混雑 × 関係性）

type InteractionCombiner =
  | "superadditive"    // f(a,b,…) > Σ（荷物+階段+混雑 が和より重い）
  | "saturating"       // 逓減（relief 源の重複は線形に積まない）
  | "gating"           // 入力が他入力の発火を条件づける（雨×屋外は fallback 有無で gate）
  | "sign_flip"        // 符号反転（静けさ=価値↔退屈）
  | "veto_escalation"  // soft 負荷が閾値超で hard block に昇格（夜の安全閾値・step-free 連続断絶）
  | "threshold";       // 非線形ステップ（X 超で効果が跳ねる）

interface InteractionTerm {
  id: string;
  scope: InteractionScope;
  inputs: StateRef[];                 // L1..L9 への型付き参照
  modifies: FitComponentKey | "hardBlock" | { construct: ConstructAxis };  // ★ §4.2
  combiner: InteractionCombiner;
  hardBlockCapable: boolean;
  confidence: "min_of_inputs" | "product_of_inputs";  // ★ §4.3 弱リンク連鎖
  visibility: Visibility;
  missingData: "no_fire" | "fail_closed";             // ★ §4.4
}
```

### §4.2 ★不変条件：相互作用は「既存 component の修飾子」であって「新たな並列スコア」にしない
`modifies` は**必ず既存の component / construct / hardBlock を指す**。相互作用は独立スコアを生成しない。これにより「同じ信号を相互作用としてもう一度加点/減点する」二重計上を**構造で**塞ぐ（§8 の上位原則）。`combiner=none` で済ますのではなく、型レベルで「修飾子であって score でない」を強制する。

### §4.3 confidence 連鎖（弱リンク）
相互作用の confidence = **入力の最弱（min）または積（product）**。高インパクトでも 1 入力が低 confidence なら相互作用自体が低 confidence → **hard verdict でなく質問**として表出。鎖は最弱の環の強さしか持たない。

### §4.4 missing-data
入力欠損時: 安全非関与 → `no_fire`（保守的に発火させない）／ 安全 critical → `fail_closed`（veto 維持）。価格/在庫は捏造しない。

## §5 多因子 valence ―― recoveryStyle 単独でなく 8 因子

`valence(indicator) = f(recoveryStyle, tripIntent, relationship, timeOfDay, fatigue, role, phase, context)`

| 因子 | 役割 | 符号変化例 |
|---|---|---|
| recoveryStyle | 静養/刺激で快不快反転 | crowd・soundscape・静けさ |
| tripIntent | recovery/work/social/exploration/romance | workSuitability・connectivity の重要度 |
| relationship | solo/family/romance/friends/colleagues | nighttimeSafety・childFriendliness・privacyForTwo |
| timeOfDay | **昼夜分離（平均禁止）** | safety 体感・soundscape・lighting |
| fatigue | participantState 疲労 | physicalEffort・waiting の負荷係数 |
| role | 何として扱うか（L3） | 同一対象でも目的次第で再評価 |
| phase | 計画/移動/滞在/後（fatigue-compounding） | bookingRigidity・connectivity・累積疲労 |
| context | FitContext(L6) | 季節・天候・同行で全体補正 |

固定方向を持たない indicator（novelty/crowd/safety-edgy/静けさ）が多数 → `VALENCE_FLIP` は §4 の sign_flip 相互作用で per-user 反転。**昼夜は常に別 indicator**。

## §6 多因子 rollup ―― tripIntent 単独でなく 7 因子

`rollup(score) = g(tripIntent, objectRole, category, FitContext, participantState, relationship, travelPhase)`

| 因子 | rollup への効き |
|---|---|
| tripIntent | `CONTEXT_ROLLUP_OVERRIDE[work]` で workSuitability/connectivity を強く集約 |
| objectRole(L3) | role-affinity が族の重みを再配分 |
| category(L4) | 9 カテゴリで rich-facet の重み構成が異なる |
| FitContext(L6) | 季節/天候/予算/同行で全体補正 |
| participantState | 疲労/障害/妊娠/年齢で burden 族の重み増 |
| relationship | safety/child/conversation の重み再配分 |
| travelPhase | phase 毎に connection/booking の効きを変える |

gate-first 維持: L5 veto → L5r relational → 加重集約 → confidence 添付（veto 通過まで score を出さない）。

## §8 二重計上禁止マトリクス

| 禁止ペア | 分離規則 | 扱い |
|---|---|---|
| crowd-as-value vs crowd-as-burden | role/recoveryStyle で**一方**に符号確定 | `IX_crowd_role` が sign_flip・合算しない |
| priceValue vs budgetFit | 対価妥当性(質) / 予算適合(L6 文脈) | 別レイヤ別計上・価格捏造禁止 |
| aestheticRefinement vs hygieneCleanliness | **古い≠汚い**・surfaceWornVsDirty で分離 | `IX_hygiene_budget` combiner=none |
| smellAirComfort vs hygieneCleanliness | 臭気は衛生の一症状 | smell は別 indicator・衛生に畳まない |
| perceivedSafety vs crisisRobustness | 体感(L1b) ≠ 構造リスク(L2) | 別 construct |
| digitalConnectivity vs workSuitability | 接続 ≠ 就業適性 | 別 construct・IX で連結のみ |
| conversationMealFit vs workSuitability/ritualLoad | 会話の質 ≠ 通話/集中 ≠ 作法負荷 | 三者分離 |
| gradedAccessibilityComfort vs L5 access veto | 代償帯 ≠ 絶対 veto | L2 と L5 を跨いで二重に効かせない |
| noveltySeeking(trait) vs localAuthenticity(indicator) | trait は符号を**反転するのみ**・自身を score に足さない | flip only |
| **相互作用 vs 素の component**（§4.2） | 相互作用は既存 component の**修飾子**・新スコアを作らない | 型で `modifies` を既存に限定 |

## §9 欠損挙動マトリクス

| データ種別 | 欠損時挙動 | confidence | 質問化 | 捏造 |
|---|---|---|---|---|
| safety-critical（allergen・車椅子連続性・安全閾値・災害） | **fail-closed**（veto 維持/安全側除外） | — | 高優先確認 | 禁止 |
| 通常 indicator（雰囲気/衛生体感/言語/接続） | confidence 減 + 必要なら 1 問で観測 | 下げる | する | しない |
| price / availability | **絶対に捏造しない**・欠損保持・未確定表示 | 影響小 | 確認導線 | **厳禁** |
| trait（novelty/pace 等 user 軸） | 中立扱い（符号反転を発火させない）+ compact 質問 | 中立 | する | — |
| provenance(L8) | source-count は **confidence のみ**に作用・生スコア不変 | source 多=確度↑ | — | — |

原則(再掲): source 数は confidence にのみ効き生品質スコアを動かさない（出典が多い=高評価、ではない）。

## §10 実装推奨 ―― 次バンドルの選択

| 案 | 利点 | 欠点 |
|---|---|---|
| **A** construct registry 先行 | trait 空間が確定し全 scoring の土台・批評の欠落族(F〜I)を型で固定 | interaction 未配線では valence 反転が効かず体感薄い |
| B interaction registry 先行 | flip/gate/cascade が効き体感差 | 参照 construct 未確定だと宙に浮く |
| C ConnectionState 深掘り | L7 が固く route 強化 | 単一カテゴリ偏重・横断 trait 遅延 |
| D B2/C2/D2 を小バンドル分割 | 着地安全・段階検証 | 全体像遅延 |

### 推奨: **A（construct registry 先行）＋ D の分割規律を併用**
理由: (1) 批評の致命的欠落（F_safety/G_condition/H_communication/I_infra-work/noveltySeeking 昇格/gradedAccessibility）は全て **construct 層の穴** で、interaction(B) は存在しない construct を参照できない。(2) `as-const` typed registry を先に固めれば safety-critical 型ロックと double-count 禁止ペアの型表現（§2/§8）が後続の前提になる。(3) D の小バンドル規律は A の中に取り込む（B2/C2/D2 を一度に動かさず additive に刻む）。

### 最小 first additive bundle（pure core・配線なし・既存不変更）
1. **F_safety / G_condition の construct スケルトン**を `as-const` で追加（perceivedSafety の**昼夜分離** indicator・hygieneCleanliness の**worn/dirty 分離**を型で固定）。
2. **noveltySeeking を L1 trait に昇格**する型定義のみ（valence flip は未配線・`IX_novelty_flip`/`IX_quiet_recovery` は registry に id+combiner 宣言だけ・発火は次バンドル）。
3. **missing-data 既定を型に焼く**（safety-critical=fail-closed フラグ・provenance=confidence-only・interaction confidence=min_of_inputs）。
4. 既存 `fit-types.ts`（FitContext/RelationshipKind/ReliefAxis/HardConstraint）には**触らない**。新規 registry は additive・escape hatch(`ext`) は宣言のみ。

→ 「construct 層の穴」を最小・安全・additive に塞ぎ、後続で interaction(B) 配線・ConnectionState(C) 深掘りを刻んで載せる。

## §12 guardrail / 不変条件

- pure・additive・未配線。実 API/booking/price 断定/永続化/UI/solver/M2 runtime/Plan Intelligence/push なし。
- **非 opaque**: rollup 重み・combiner・VALENCE_FLIP・写像表を export。
- **相互作用は修飾子**（§4.2）・新並列スコアを作らない（二重計上を型で塞ぐ）。
- **confidence 連鎖**（§4.3）: 相互作用は最弱入力の confidence を継ぐ。
- provenance: 指標ごと・source は confidence のみ・生スコア不変。
- safety-critical 欠損=fail-closed・price/在庫=捏造禁止・通常欠損=confidence 減+質問。
- private 非漏洩（構築子/指標/相互作用も visibility を持ち shared 射影で削除）。
- authority: fit は scoring/説明のみ・`authoritative=false`・`hasFitActionAuthority` literal false。
