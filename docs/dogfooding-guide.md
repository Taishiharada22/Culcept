# Stargazer Layer 3 — 創業者ドッグフーディングガイド

## 概要
「今日の自分」（`/stargazer/engine`）を3日間実際に使い、Layer 4 に進むべきか判断する。

## 毎日やること（CEO）

### 朝（5分）
1. `/stargazer/engine` を開く
2. **Daily Intervention** カードの内容を読む（今日の状態推定）
3. **Decision Engine** でプリセットまたは自由入力で1つ以上相談する
4. **Self vs Oracle** で今日の予測を入力する（3問、選ぶだけ）

### 夜（3分）
1. `/stargazer/engine` を再度開く
2. **Self vs Oracle** の「答え合わせ」を完了する
3. Decision Engine の結果を振り返り、フィードバック（当たってた/ずれてた + 納得感5段階）を入力する

## テストで確認してもらうこと

### Day 1: 基本動作
- [ ] 3カードが全て表示される
- [ ] プリセット4つが全てタップ可能
- [ ] Decision Engine の結果が返ってくる（保留でもOK）
- [ ] 正確性フィードバック（3択）→ 納得感（5段階）→ お礼画面の遷移
- [ ] SvO 予測入力 → 1問ずつスライド → 確定ボタン

### Day 2: 答え合わせ + 精度感
- [ ] SvO「答え合わせの時間です」が表示される
- [ ] 答え合わせ完了後、あなた vs Oracle のスコアが表示される
- [ ] Decision Engine で「自由入力」の相談も試す
- [ ] Daily Intervention の内容が前日と違う

### Day 3: 判定
- [ ] 3日間の集計SQLを実行（`scripts/dogfooding-metrics.sql`）
- [ ] 合格判定サマリーの3指標を確認

## 合格ライン

| 指標 | 測り方 | 合格 |
|------|--------|------|
| 言い当て精度 | DE「当たってた」率 | > 50% |
| 納得感 | 5段階平均 | >= 3.5 |
| 継続意欲 | 3日中のアクセス日数 | >= 2日 |

## 合格後
→ Layer 4（Early Warning）の実装に進む

## 不合格の場合
→ Layer 3 の精度改善を優先（AI出力チューニング / プロンプト改善 / 状態推定精度向上）

## 集計方法
Supabase SQL Editor で `scripts/dogfooding-metrics.sql` を実行。
最後のクエリ「合格判定サマリー」が3指標のpass/failを表示する。

## UXの気になる点
ドッグフーディング中に気づいた違和感・邪魔な点は都度メモしてください。
修正は3日間の集計後にまとめて行います。
