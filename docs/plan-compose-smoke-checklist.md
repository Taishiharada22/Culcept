## 0. 前提（compose UI を表示する）
1. `.env.local` に **`PLAN_COMPOSE_TIMELINE_ENABLED=true`** を追加（CEO が設定）。
2. dev 再起動（`npm run dev`）。
3. `/plan` を開く → 予定追加で**2カラムの compose sheet**（左タイムライン＋右フォーム）が出れば OK。
   - 出ない場合: env が読まれていない（再起動忘れ）か、別タブ。`shouldUseComposeSheet` が gate（`PlanClient.tsx:947`）。

> flag OFF のままだと旧 UI になり、以下は検証できません。必ず ON を確認してから開始。

## 1. #14 二重登録（最優先・データ整合）
- 手順: 予定追加 → 「なにをする」「どこで」入力 → 左タイムラインに配置 → **青の「完了」** → 一覧/タイムラインに **1 件だけ**。
- 再オープン: もう一度「予定追加」を開く → さっきの予定が **二重に出ない**（1 件）。
- PASS 条件: 追加が 1 件・再オープンで重複なし。Network で POST が 1 回。

## 2. #15 重なり（2.5D layered ＋ 段差rail）
- 手順: 「仕事 9:00–18:00」（長時間 context）＋「会議 10:00–11:00」＋「会議 14:00–16:00」を配置。
- PASS 条件: 仕事＝**背景バンド（淡い）**、会議＝**前面ブロック**。3 重以上でも **どこからどこまでか**が段差rail で読める。範囲が潰れない。

## 3. カテゴリー配色（種別が1目で）
- 手順: 朝食/ランチ・会議・ジム・仕事 を配置。
- PASS 条件: food=amber / meeting=indigo / fitness=emerald / work=violet帯 で **色だけで種別が即判別**。**背景帯は前面より淡い**同系色。

## 4. #16 6時以前（適応窓）
- 4-1 早朝予定あり: **4:00 開始**の予定がある日 → タイムラインが **4:00 から始まり全可視**（6:00 で切れない）。最上端＝4:00。
- 4-2 6時以降のみ: 早朝予定が無い日 → **6:00 開始のまま**（表示・配置・高さが従来と不変）。
- 4-3 drag 非ジャンプ: 配置済み予定をドラッグ移動 → 窓が途中で**伸縮せず指と時刻がズレない**。離した後に早朝へ動かしていれば窓が適応。
- 4-4 早朝入力: 時間ホイールで予定を 4:30 等に変更 → **編集中もその予定が見える**（窓が適応）。
- PASS 条件: 4-1〜4-4 すべて。

## 5. compose / 編集フロー
- 5-1 再編集: 配置済み予定をタップ → 編集バー → 時間/場所変更 → 編集バーの **「完了」は編集のみ確定**（フロー全体は完了しない）。最終確定は **青の「完了」**。
- 5-2 既存編集: 既存予定をインライン編集（amber バー）→ 完了/キャンセルが効く。編集中の元ブロックと二重表示しない。
- 5-3 場所候補: 「なにをする」入力後に横の SVG をタップ → その予定に応じた **よく行く＋最近** が出る（タイトル連動）。1 タップで「どこで」入力、長押しで詳細。
- PASS 条件: 「完了」の二段（編集のみ／最終）が分離・二重表示なし・候補が機能。

## 記録方法
各項目を ✅PASS / ⚠️要修正 / ❌NG で記録。⚠️/❌ は「画面・操作・期待と実際」を1行で残す → こちらで最小修正 → 再 smoke。
全 PASS で各 task を **product complete** に更新（それまでは smoke pending）。
