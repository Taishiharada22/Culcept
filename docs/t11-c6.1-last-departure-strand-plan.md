## §1 前提を疑う — 次は last_departure_strand で正しいか

| 候補 | 評価 |
|---|---|
| **IX_last_departure_strand** | **★ 採用**。C5 RouteReliabilityState+last_departure_lock を消費する**純粋 fit/risk**(終電逃しリスク・scheduling でない) |

**推奨 = last_departure_strand**。純粋(booking/scheduling なし)・C5 state を消費・現実の旅行リスク（終電/最終便逃し→足止め）を fit に反映（directive ⑤）。cancel_weather は T6 owning phase 待ち。

### grounding（reliability 理論）
- **FHWA Travel Time Reliability**: Buffer Index =(95%ile−mean)/mean = 計画に足す余裕。Planning Time Index =95%ile/free-flow。**遅延が buffer を超えると約束時刻を逃す** = strand の本質。
- **GTFS transfer_type**: timed(1)/min-time(2) 乗換は遅延接続リスク。最終便直前の脆弱な乗換は strand を増幅。
- 原理: `strand risk = 遅延が余裕(buffer)を食い潰す確率`。終電 lock があり buffer が薄く delay risk が高い時、足止めが起きる。fragility(乗換/egress)が増幅・fallback が緩和。**実時刻/時刻表を断定しない**(推定 risk のみ)。

## §3 strand risk モデル（★delayRisk≠PTI 分離で二重計上を構造回避）

```
last_departure_lock 無 → 非発火（strand=0）
strandBase = delayRisk × (1 − bufferIndex)                       // 遅延 × 余裕不足（buffer 理論）
strand     = clamp( strandBase × (1 + transferFragility + egressFragility) × (1 − fallbackAvailability×0.5) )
```
- **gating**: last_departure_lock が gate（lock 無→発火しない）。
- **threshold/saturating**: 高 delay × 低 buffer で strand 急増、fragility で増幅、fallback で緩和。
- 係数（fragility 重み・fallback 緩和率・floor）は **export（非 opaque）**。
- ★ **delayRisk/bufferIndex/fragility を strand に使い、planningTimeIndex は routeChainBurden(C5 burden)に使う** = 同一 reliability の **time-burden 面**と **strand-risk 面**を別 field・別 component に分離（§7 二重計上防止の核）。

## §4 効果（risk/labelCap が主・burden 二重計上しない）

```
strand > STRAND_VETO_FLOOR(例0.6)   → labelCap="stretch"（脆い route を top rec にしない）+ riskFlag "last_departure_strand_high"
STRAND_CAUTION_FLOOR(例0.3) < strand → labelCap="good" + riskFlag "last_departure_strand_caution"
それ以下                            → 効果なし
```
- **riskFlag / labelCap が主**（routeChainBurden=時間負荷とは別の「足止めリスク」consequence）。
- burdenFit は **原則修飾しない**（二重計上回避）。明示正当化がある場合のみ軽微 stress penalty(optional・既定オフ)。
- **実 last train/flight availability を断定しない**・**itinerary を solve/order しない**・**booking/scheduling authority を作らない**（lock/reliability 状態を**読む**だけ）。

## §6 privacy

- private mobility/anxiety/fatigue 懸念（例: 終電を逃す不安が強い user）は **full に効くが shared に出さない**（C3/C4/C5/C6 two-layer 踏襲）。private な strand 懸念 → full-only labelCap/riskFlag（private）。
- shared 射影は shared-safe interaction 出力から再導出（`availableShared`/shared-safe confidence/signalBasis/labelCap）。
- **confidence/available/signalBasis/reason/rationale/riskFlag に private strand 懸念の存在を漏らさない**（canary）。
