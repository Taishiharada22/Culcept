# ZOZOTOWN / WEAR JTBD比較分析 vs. Personal Style OS

## リサーチレポート: ファッションプラットフォームの Jobs to Be Done 比較
日付: 2026-04-03

### 要約（3行以内）

ZOZOTOWN/WEARはEC転換率と「似合う発見」を軸にAI投資を加速しているが、ユーザーの「手持ちの服を活かす日常判断」には踏み込めていない。日本市場でクローゼット管理+日常コーデ判断+スタイル自己理解を統合するプレイヤーは不在。Personal Style OSが独占可能な領域は「購買前・購買後の日常的なスタイル意思決定」にある。

**計測テクノロジー群（ハード）**

**AIパーソナライゼーション（ソフト）**
- ホーム画面の完全パーソナライズ（購買履歴+閲覧行動+天候+季節）

- 得た知見をWEARアプリのジャンル診断に還元

**LYST買収（2025年4月、$154M）**
- グローバルファッション検索プラットフォーム
- 年間1.6億人のショッパー × 27,000ブランド
- LLMベースのファッション特化検索/ディスカバリーをAI強化方針

### UX上の強み
1. **計測→購買の直結**: 身体データが「合うサイズ」に直結する明確なバリュー
2. **データ量の圧倒的優位**: 1,400万件のコーデデータは日本最大
3. **「似合う」のデータ化**: ジャンル診断→パーソナライズ推薦の一気通貫
4. **購買体験の最適化**: あくまでEC転換率向上がKPIの中心

### 構造的制約
- **全てがEC導線に帰着する**: 「似合う」も「ジャンル診断」も最終的にZOZOTOWN商品購入が出口
- **手持ち服の管理機能なし**: クローゼット内のアイテムを扱う機能は一切ない
- **日常のコーデ判断を支援しない**: 「今日何を着る？」に対する回答は提供しない
- **天気・スケジュール・気分との統合なし**: 購買コンテキストは扱うが、生活コンテキストは扱わない

## 2. WEAR by ZOZO 現状分析（2025-2026）

### 基本データ
| 指標 | 値 |
|------|-----|
| 累計DL | 1,800万以上 |
| コーディネート投稿数 | 1,400万件以上 |
| コアユーザー層 | 10代後半～30代前半（Z世代中心） |
| ポジショニング | 日本最大のファッションコーディネートSNS |

### 主要機能（2024-2025リニューアル後）

**コーディネート検索・閲覧（メディア機能）**
- 1,400万件のコーデから検索・フィルタリング
- 体型絞り込み検索（身長・体型が近い人のコーデ発見）
- WEARISTAなどインフルエンサーのコーデ参照

**AIパーソナライズ（2024年5月リニューアル）**
- ファッションジャンル診断: 好みの画像5枚以上選択→12ジャンルの配合を円グラフ表示
  - 12ジャンル: ガーリー/フェミニン/ヘルシー/きれいめ/マニッシュ/ナチュラル/ラフ/シンプル/モード/ポップ・ユニーク/ストリート/アウトドア・スポーツ
- ホーム画面のパーソナライズ（診断結果+閲覧履歴ベース）
- 「似合う」を構成する4要素: ジャンル / 味付け / 与えたい印象 / 体型の悩み

**着回し提案（2025年10月導入）**
- AIが1,400万件のコーデから着回しパターンを提案
- 「体型の悩み」登録でパーソナライズ精度向上
- ZOZOTOWNとの連携: 気になるアイテムをそのまま購入可能

**EC連携機能**
- コーデ内アイテムのZOZOTOWN直接購入
- WEARからZOZOTOWNへのシームレス遷移

**SNS機能**
- コーデ投稿・保存・フォルダ管理
- ランキング・フォロー
- フルメイクAR（お試しメイク）

### UX上の強み
1. **コーデ参照の圧倒的データ量**: 1,400万件は他の追随を許さない
2. **「自分と似た人」の発見**: 体型・身長フィルターによる「自分ごと化」
3. **AIジャンル診断の手軽さ**: 画像選択のみで好みを数値化
4. **ZOZOTOWN連携**: コーデ→購買の滑らかな導線

### 構造的制約
- **他人のコーデ起点**: 自分のクローゼット内のアイテムからコーデを組む機能はない
- **「見る」が主体**: インプット（他人のコーデを見る）中心で、アウトプット（今日自分が着るものを決める）を直接支援しない
- **EC導線が最終ゴール**: ZOZOTOWN商品購入が収益モデルの根幹
- **天気・スケジュール・気分との統合なし**
- **スタイル成長の可視化なし**: ジャンル診断は1回完結。時系列でのスタイル変化や深化は追えない
- **ユーザー不満**: 2024年アップデートでお気に入り操作の煩雑化、写真表示サイズの縮小が指摘

## 4. Jobs to Be Done 比較マトリクス

### JTBD定義

ユーザーがファッションに関して「雇う」仕事を8つに分類する。

| # | Job to Be Done | 略称 |
|---|----------------|------|
| J1 | 「自分に似合うものを知りたい」 | 似合う発見 |
| J2 | 「今日何を着るか素早く決めたい」 | 日常コーデ判断 |
| J3 | 「手持ちの服を把握・管理したい」 | ワードローブ管理 |
| J4 | 「天気や予定に合った服を選びたい」 | コンテキスト適応 |
| J5 | 「自分のスタイルを理解し成長させたい」 | スタイル自己理解 |
| J6 | 「おしゃれな人のコーデを参考にしたい」 | コーデ参照 |
| J7 | 「自分に合うサイズの服を買いたい」 | フィット購買 |
| J8 | 「気分に合った服で自分を表現したい」 | 感情表現 |

### 比較スコアリング（5段階: -- / - / 0 / + / ++）

| JTBD | ZOZOTOWN | WEAR | XZ | Personal Style OS |
|------|----------|------|----|-------------------|
| J1: 似合う発見 | ++ | + | 0 | ++ |
| J2: 日常コーデ判断 | -- | - | + | ++ |
| J3: ワードローブ管理 | -- | -- | + | ++ |
| J4: コンテキスト適応 | - | -- | + | ++ |
| J5: スタイル自己理解 | 0 | 0 | -- | ++ |
| J6: コーデ参照 | + | ++ | 0 | 0 |
| J7: フィット購買 | ++ | + | -- | 0 |
| J8: 感情表現 | -- | -- | -- | ++ |

### 解説

**ZOZOTOWN が強い領域（J1, J7）**
- 計測テクノロジー（ZOZOMAT/ZOZOGLASS/ZOZOSUIT）＋1,400万件コーデデータで「似合う」と「合うサイズ」をデータ化する力は圧倒的
- ただし全てがEC購買に帰着する。「似合う」の発見は「次に買うもの」の文脈でのみ機能

**WEAR が強い領域（J6）**
- 1,400万件のリアルコーデデータは唯一無二。「自分と体型が似た人のコーデ」発見力が高い
- しかし「見る」が主体であり、「今日自分が着るものを決める」行動には直結しない

**XZ が強い領域（J2, J3, J4）**
- 手持ち服×天気の統合は日本市場でほぼ唯一の存在
- ただし有料化（月200円）によるユーザー離れ、スタイル理解機能の欠如が弱点

**Personal Style OS が独占可能な領域（J2, J3, J4, J5, J8）**
- **5つのJTBDで競合不在または弱い状態**。特にJ5（スタイル自己理解）とJ8（感情表現）は誰もまともに取り組んでいない
- J2+J3+J4の統合（日常判断×管理×コンテキスト）もXZ以外に存在せず、XZは自己理解レイヤーがない

## 5. 競合が踏み込めない構造的理由

### ZOZOTOWNがPersonal Style OS領域に来ない理由

1. **収益構造の制約**: ZOZOのGMV 6,000億円はEC手数料モデル。ユーザーの手持ち服を活かす=購買減少に直結するため、構造的に手持ち服起点のサービスを作るインセンティブがない
2. **KPIの不一致**: 「購入金額2倍」が似合うラボの成功指標。ユーザーの日常コーデ判断支援は購入頻度と無関係または負の相関
3. **データの方向性**: ZOZOのデータは「何を買ったか」「何を見たか」。「何を持っているか」「何を着たか」のデータは持っていない
4. **LYST買収の示す方向**: グローバル「購買ディスカバリー」への投資。生活コンテキスト側ではなく購買コンテキスト側への拡張

### WEARがPersonal Style OS領域に来ない理由

1. **SNSモデルの制約**: WEARは投稿×閲覧×フォローのSNS構造。個人のクローゼット管理は「孤独な作業」であり、SNSのエンゲージメントモデルと相性が悪い
2. **EC連携がコア収益**: WEARの価値はZOZOTOWNへの送客。手持ち服活用はEC送客と相反する
3. **データ構造の違い**: WEARのデータは「他人のコーデ写真」。自分のワードローブデータとは根本的に異なる
4. **着回し提案の限界**: 2025年導入の着回し提案は「他人のコーデから着回しパターンを提示」であり、「あなたのクローゼットの中から今日の最適解を出す」とは全く異なる

### XZがPersonal Style OS領域に満たせない理由

1. **自己理解レイヤーの欠如**: 天気×手持ち服のコーデ提案はあるが、「なぜこのスタイルを選ぶのか」「自分のスタイルDNAは何か」への回答がない
2. **感情・気分の統合なし**: 天気・気温は外部コンテキストだが、内部コンテキスト（気分・エネルギー・与えたい印象）は扱わない
3. **スタイル成長の可視化なし**: 過去の選択パターンからのインサイト抽出、スタイル変化の時系列可視化がない
4. **規模の壁**: 有料化後のユーザー離れが示すように、単体クローゼット管理アプリの成長に限界がある

## 6. Personal Style OS が独占すべき領域

### ポジショニング定義

```
ZOZOTOWN = 「何を買うか」を最適化するプラットフォーム（EC）
WEAR     = 「何が流行っているか」を発見するプラットフォーム（メディア/SNS）
XZ       = 「天気に合う服」を提案するツール（ユーティリティ）
---------------------------------------------------------------------------
Personal Style OS = 「自分はどういうスタイルの人間か」を理解し
                    「今日の自分にとって最適な装い」を日常的に判断する
                    第二の自己としてのOS（自己理解×意思決定支援）
```

### 独占可能な4つの価値領域

**領域1: ワードローブ・インテリジェンス**
- 手持ちアイテムの把握+分析+Gap検出
- 「持っているのに活かせていないアイテム」の発見
- カテゴリ・色・テイストの偏り可視化
- 競合状態: ZOZOTOWN=なし、WEAR=なし、XZ=管理のみ（分析なし）

**領域2: コンテキスト統合型コーデ判断**
- 天気 + スケジュール + 気分 + エネルギー + 与えたい印象の5軸統合
- 「今日の自分」に最適な手持ち服の組み合わせを提案
- 競合状態: ZOZOTOWN=なし、WEAR=なし、XZ=天気のみ

**領域3: スタイル自己理解の深化**
- 選択パターンからのスタイルDNA抽出
- 「無意識に選んでいるもの」の言語化
- スタイル変化の時系列トラッキング
- 「自分って、こういうスタイルの人間だったのか」というAha Moment
- 競合状態: 全プレイヤーで不在。WEAR のジャンル診断は1回完結で深化しない

**領域4: 感情-装い接続**
- 気分・心理状態と服装選択の関係性を観測
- 「こういう気分のときにこの服を選ぶと調子がいい」のパターン発見
- Stargazerの深層観測データとの統合（判断特性×スタイル選択）
- 競合状態: 全プレイヤーで完全に不在

### 競合が模倣困難な理由（MOAT）

1. **Stargazer統合**: 45軸の性格・判断特性データとスタイルデータの掛け合わせは、ファッション企業にもAIチャット企業にも再現困難
2. **「買わない」がゴール**: EC企業は構造的に「手持ち服を活かす」方向に投資できない。これは競合不在の最大要因であり、最大のビジネスモデル課題でもある
3. **日常接点の蓄積**: 毎日の「今日何着る？」は高頻度接点。ECは購買時のみ、SNSは閲覧時のみ
4. **内面×外見の統合**: 気分・エネルギー・判断特性とスタイル選択の接続は、既存プレイヤーの持つデータでは実現不可能

## 7. インサイト

- ZOZOの「似合う」戦略は全てEC転換率向上に帰着する。似合うラボの「購入金額2倍」という成功指標がそれを如実に示す。このKPI構造がある限り、ZOZOは「手持ち服を活かす」方向に進めない
- WEARのAI着回し提案（2025年10月）は最も近い脅威に見えるが、実態は「他人のコーデDBから着回しパターンを見せる」であり、「あなたのクローゼットから今日の1着を選ぶ」とは根本的に異なる
- XZの有料化（月200円、2024年3月）とそれに伴うユーザー離れは、「クローゼット管理単体」では課金が困難であることを示す。自己理解やスタイル成長という付加価値なしには、ユーティリティの域を出ない
- グローバルでは Indyx / Whering / Fits などが進化しているが、いずれも「自己理解」「感情統合」「性格×スタイル」の深度はなく、あくまで「管理+提案」ツールに留まる
- **最大の空白地帯**: 「なぜ自分はこの服を選ぶのか」「自分のスタイルはどう変化しているか」「今の気分に最も合う装いは何か」を統合的に扱うサービスは、日本にもグローバルにも存在しない

## 8. 推奨アクション

1. **Calendar機能のポジショニング明確化**: ZOZOTOWN/WEARと戦わない。「手持ち服 × 天気 × 気分 × スケジュール × 性格」の5軸統合を前面に出し、EC導線とは明確に異なることを打ち出す
2. **XZとの差別化**: XZは「天気×手持ち服」の2軸。Aneurasyncは「天気×手持ち服×気分×スケジュール×スタイルDNA」の5軸。この差が「ツール」と「OS」の違いになる
3. **WEARの「着回し提案」への対抗**: 「他人のコーデを参考にする」（WEAR）ではなく「自分のクローゼットから自分の気分に合う装いを選ぶ」（Aneurasync）という明確な対比を意識する
4. **Stargazer × Calendar 統合の優先度引き上げ**: 性格データ（判断特性、感情パターン）× スタイル選択データの接続は、全競合に対する唯一のMOAT。この統合が遅れると「便利なクローゼットアプリ」に縮退するリスクがある
5. **「買わない」の価値言語化**: ECがゴールでないことは弱みに見えるが、サステナビリティ文脈では強み。「手持ちの服をもっと活かす」「自分のスタイルを深く知ることで無駄な買い物を減らす」というナラティブは、Z世代の価値観と強く共鳴する可能性がある
