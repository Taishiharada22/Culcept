## 1. 価格設計

### 1.1 価格設定力の根拠

Aneurasyncの価値は**時間とともに複利で増える**:
- マッチングアプリ: 使い続けても価値が一定
- 結婚相談所: カウンセラーの記憶に蓄積されるが退会で消える
- Aneurasync: 使うほどAlterが深くなり、Counselorの判断精度が上がる

**スイッチングコストが極めて高い＝強い価格設定力。**

Aneurasyncの比較対象は結婚相談所だけではなく、エグゼクティブコーチング・心理カウンセリング・プレミアムメンタルヘルスサービスも含む。

### 1.2 料金体系

| **Core** | 4,980円 | Stargazer全Phase、Alter Deep（深層対話・判断エンジン・Daily Guidance）、Calendar/コーディネーター、Genome Card完全版、既知ペア無制限＋全インサイト＋累積分析、Friendship community/business | — |

### 1.4 成婚料のバイアス相殺設計

成婚料を設定するが、「成婚に導く圧力」を構造的に相殺:

- Counselorの内部評価指標: 「成婚数」ではなく「関係の確信度 × ユーザー満足度」
- 「この関係は進めない方がいい」判断にペナルティなし
- 成婚料の説明: 「成婚した事実」ではなく「成婚に至るまでの全プロセスの価値」
- **Honest Exit Rate（正直な撤退率）を公開指標にする** → 撤退を勧めることが企業価値になる構造

### 1.5 Free枠の戦略的意味

既知ペア1組を無料にする理由:
- 招待ループの起爆剤（1組だけ無料→驚くほど深い関係性インサイト→もっと試したくなる→Core課金）
- Stargazer Phase 2到達まで無料 = 自己理解の価値を体感してからRendezvousに進む導線
- 否定のティーザー表示（「ここに気づきがあります」）で課金動機を自然に作る

### 1.6 Partner課金開始タイミング

```
1. Partner登録（無料）→ 待機状態
2. その人にとってのマッチング候補がN人以上揃う
   → 「あなたに合う候補が見つかりました」通知
   → 初回課金開始
3. MVP段階: シンプルに「Partner登録50名到達で全員に開始通知」でも可
```

「紹介できる相手がいないのに課金される」という不信感をゼロにする。

## 2. 本人確認・書類認証

### 2.1 段階的認証

| 対象 | 必要書類 |
|------|---------|
| 全員（18歳以上） | 顔写真付き本人確認、年齢確認、電話番号 |
| Partner参加時 | + 独身証明書（本籍地の市区町村発行）、現住所証明（住民票） |
| 年収を表示する人 | + 収入証明書（源泉徴収票、確定申告書等） |
| 学歴/資格を表示する人 | + 卒業証明書・資格証明書 |

### 2.2 「認証済みだが非表示」オプション

収入証明を提出するが、具体的な年収は相手に非表示——「収入認証済み」バッジだけ表示。

**理由:**
- 嘘がないことは保証しつつ、条件フィルタリングの弊害を減らす
- 「条件ではなく判断原理で出会う」というAneurasyncの思想と整合
- リサーチで「年収で足切りされる」が男性の大きなペインとして浮上
- 表示/非表示はユーザーの選択

### 2.3 設計根拠

> 本人確認だけではアプリとの差が出ない。Rendezvousが「結婚相談所の完全上位互換」を名乗るなら、独身証明が信頼の中核。（CEO）

IBJの公式: 独身証明書、収入証明書、本人確認書類、現住所証明書、卒業証明書、資格証明が入会時の標準書類。Rendezvousはこれを踏襲しつつ、必須/任意の段階を設けて摩擦を最小化。

## 3. 法規制対応

### 3.1 特定商取引法（特定継続的役務提供）

**適用条件:** 契約期間2ヶ月超 + 契約金額5万円超
- Partner/Premium: 月額×継続期間で超過するため適用対象
- Romance/Core: 月額制で短期解約可能だが、実質継続前提のため適用対象として設計するのが安全

**事業者の義務:**
- 概要書面の交付（契約前）
- 契約書面の交付（契約締結後遅滞なく）
- クーリングオフ: 契約書面受領日から8日間（電子通知も可、2021年改正）
- 中途解約: いつでも可能。違約金上限あり（開始前3万円、開始後は提供済み対価＋2万円or残額20%の低い方）
- 誇大広告の禁止
- 不実告知の禁止

### 3.2 個人情報保護法

**AIプロファイリング関連:**
- AIプロファイリングを行う旨を利用目的に明示する義務
- 推知情報（AIが推定した性格特性、判断パターン等）の取扱いポリシー明示
- 性的指向を推知しうるアルゴリズムは特に慎重に（要配慮個人情報に準ずる扱い）
- プロファイリング規制は将来強化の見込み → プライバシーバイデザインで先行投資

**未成年:**
- 16歳未満: 親権者の同意が必須
- 未成年データは別のストレージポリシー
- 18歳到達時にデータ継続利用の再同意

### 3.3 景品表示法

- 成婚率等の表示: 算出方法を必ず明記
- 6指標体系の全てに計算方法を公開 → 業界の不透明性と差別化
- AIマッチングの「精度」「成功率」等の表現は根拠データ付きでのみ使用

### 3.4 出会い系サイト規制法

- 18歳未満の利用禁止（Rendezvousの新しい出会い機能）
- 既知ペアの関係性インサイトはマッチングではないため対象外
- 年齢確認の義務（全ユーザー）

## 4. IMS認証（マル適マーク）取得計画

### 4.1 優先度: 中の上

### 4.2 実行順序

```
Step 1: 特商法対応（概要書面・契約書面・解約・料金表示を整える）
  → Partner本番化の前提条件

Step 2: Partner最小導線を本番化

Step 3: IMS申請期間を確認し、次の申請期間に間に合うよう準備
  → 申請期間が年度ごとに限定（直近は2026年1月-3月）
  → 次の申請期間を逃すと1年待つ

Step 4: インターネット型IMS認証を申請
  → 認証料金: 2年で10.5万円（古い資料ベース、要再確認）
  → マークはHP・契約書・広告物に表示可能
```

### 4.3 事前準備（Step 1と並行で進められること）

- IMS認証要件の詳細洗い出し
- Partner本番化の設計にIMS要件を織り込む（申請時の手戻り防止）
- 特商法の概要書面・契約書面テンプレートの法務レビュー

## 5. 競合比較表

### 5.1 構造比較

| 軸 | 既存結婚相談所 | マッチングアプリ | Rendezvous |
|---|---|---|---|
| 理解の源泉 | 面談メモ＋自己申告 | プロフィール＋行動ログ | **Alter（連続的深層観測）** |
| マッチング根拠 | 条件＋カウンセラーの勘 | 条件＋アルゴリズム | **判断原理・揺れ方・エネルギーパターンの相性** |
| フィードバック | 自己報告→カウンセラー変換 | なし（既読スルーが全て） | **行動観測＋自己認識ズレの可視化** |
| 関係構築 | お見合い（1時間カフェ） | メッセージ→デート | **チャット→電話→対面（段階的・観測付き）** |
| 温度差検知 | カウンセラーの聞き取り | 不可能 | **双方の行動データ同時分析** |
| 撤退判断 | なし（成婚バイアス） | 自己判断のみ | **Counselorが正直に評価** |
| 自己理解 | 副次的（面談時間の制約） | なし | **中核機能** |
| 24時間対応 | 不可能 | セルフサービス | **AI Counselor常時稼働** |
| 既存関係の理解 | 範囲外 | 範囲外 | **起動点** |
| データの蓄積 | カウンセラー退職で消失 | アカウント削除で消失 | **Alterに永続蓄積** |
| ネットワーク効果 | なし | あり（量のみ） | **あり（量＋質：関係性データが精度を上げる）** |

### 5.2 海外先行事例

**Three Day Rule「Tai」（米国, 2025年10月ローンチ）:**
- マッチメーカー15年分のデータで訓練されたAI
- 月額$25、約100問の深掘りインタビュー
- AIベッティドマッチ、リアルタイムコーチング

**Aneurasyncとの差:**
- Tai: 自己申告の100問が観測データの全て
- Aneurasync: 行動観測からの深層推定 + 矛盾検出 + 状態変化追跡 + 既知ペアの関係性データ
- 観測の質が根本的に異なる

## 6. ユーザージャーニー（理想形）

### 6.1 成年ユーザー

```
1. 友人がGenome Cardを共有してきた → 「自分のも作ってみたい」
2. Stargazerで観測開始 → Phase 1完了
   → 「自分って、そういう人間だったのか」
3. 招待トークン（ポイント）を受け取る / 自分も友人を招待する
4. 既知ペアの関係性インサイト（Free: 1組、肯定全文＋否定ティーザー）
   → 「なぜこの人と居心地がいいか、初めてわかった」
   → 否定ティーザーが気になる → Core課金
5. Core: 既知ペア無制限＋全インサイト＋累積分析
   → Phase 2到達 → 「あなたの相性パターン仮説」が出る
   → 「まだ出会っていない合う人がいるかもしれない」
6. Phase 3到達 → Romance/Friendshipで新しい人とテキスト接続
7. Counselorが行動観測フィードバック → 「自分のパターンが見える」
   → 自己認識と実際のズレが可視化される
8. チャット → 電話 → 対面 → 交際
9. （Partner希望者）書類認証 → Counselor Exchange Protocol → 深い戦略支援
10. 成婚 or 撤退 — どちらもCounselorが正直に支援
```

### 6.2 未成年ユーザー（13-17歳）

```
1. 友人がGenome Cardを見せてくれた → 「自分のも欲しい」
2. Stargazerで観測開始（親権者同意後）
3. Genome Card生成 → 友人と交換
4. 招待トークンで友人との相性インサイト
   → 「なぜこの友達とは話しやすいのか」「なぜあの子とは少しぎこちないのか」
5. Origin/Calendarで日常の行動パターン記録
6. 18歳到達 → データ継続同意 → Rendezvous全機能が解放
   → この時点でAlterの観測データが数年分蓄積済み → 精度が桁違い
```
